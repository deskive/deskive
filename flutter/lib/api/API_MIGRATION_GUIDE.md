# API Migration Guide: Moving to Dio-based API Client\n\nThis guide explains how to migrate from direct HTTP calls to the new comprehensive Dio-based API client system.\n\n## Overview\n\nThe new API client system provides:\n- **Centralized Configuration**: Single place to manage base URL, timeouts, and headers\n- **Automatic JWT Authentication**: Tokens are automatically included from AuthService\n- **Comprehensive Error Handling**: Standardized error responses with proper status codes\n- **Request/Response Logging**: Built-in debugging support in development mode\n- **Type Safety**: Strongly typed DTOs and response models\n- **Consistent API**: All services follow the same patterns and conventions\n\n## Project Structure\n\n```\nlib/api/\n├── api_services.dart                 # Export file for easy imports\n├── base_api_client.dart              # Core Dio configuration\n└── services/\n    ├── workspace_api_service.dart     # Workspace operations\n    ├── project_api_service.dart       # Project and task management\n    ├── calendar_api_service.dart      # Calendar events\n    ├── notes_api_service.dart         # Notes management\n    ├── file_api_service.dart          # File upload/download\n    ├── chat_api_service.dart          # Chat messages (non-realtime)\n    └── notification_api_service.dart   # Notification operations\n```\n\n## Getting Started\n\n### 1. Install Dependencies\n\nThe `dio: ^5.4.0` dependency has already been added to `pubspec.yaml`. Run:\n\n```bash\nflutter pub get\n```\n\n### 2. Import API Services\n\n```dart\n// Import all API services at once\nimport '../api/api_services.dart';\n\n// Or import specific services\nimport '../api/services/project_api_service.dart';\nimport '../api/services/workspace_api_service.dart';\n```\n\n### 3. Basic Usage Example\n\n```dart\nclass MyService {\n  final ProjectApiService _projectApi = ProjectApiService();\n  \n  Future<List<Project>> getProjects(String workspaceId) async {\n    try {\n      final response = await _projectApi.getAllProjects(workspaceId);\n      \n      if (response.success && response.data != null) {\n        return response.data!;\n      } else {\n        throw Exception('Failed to load projects: ${response.message}');\n      }\n    } catch (e) {\n      debugPrint('Error: $e');\n      rethrow;\n    }\n  }\n}\n```\n\n## Migration Patterns\n\n### Before: Direct HTTP Calls\n\n```dart\n// Old pattern\nFuture<List<Project>> getProjects() async {\n  final url = Uri.parse('$_baseUrl/projects');\n  final response = await http.get(url, headers: _headers);\n  \n  if (response.statusCode == 200) {\n    final List<dynamic> data = json.decode(response.body);\n    return data.map((json) => Project.fromJson(json)).toList();\n  } else {\n    throw Exception('Failed to load projects: ${response.statusCode}');\n  }\n}\n```\n\n### After: Using API Service\n\n```dart\n// New pattern\nFuture<List<Project>> getProjects(String workspaceId) async {\n  final response = await _projectApi.getAllProjects(workspaceId);\n  \n  if (response.success && response.data != null) {\n    return response.data!;\n  } else {\n    throw Exception('Failed to load projects: ${response.message}');\n  }\n}\n```\n\n## API Service Features\n\n### 1. Workspace Operations\n\n```dart\nfinal workspaceApi = WorkspaceApiService();\n\n// Create workspace\nfinal workspace = await workspaceApi.createWorkspace(\n  CreateWorkspaceDto(\n    name: 'My Workspace',\n    description: 'A new workspace',\n  ),\n);\n\n// Get all workspaces\nfinal workspaces = await workspaceApi.getAllWorkspaces();\n\n// Invite member\nawait workspaceApi.inviteMember(\n  workspaceId,\n  InviteMemberDto(email: 'user@gmail.com', role: 'member'),\n);\n```\n\n### 2. Project and Task Management\n\n```dart\nfinal projectApi = ProjectApiService();\n\n// Create project\nfinal project = await projectApi.createProject(\n  workspaceId,\n  CreateProjectDto(\n    name: 'New Project',\n    description: 'Project description',\n    type: 'kanban',\n  ),\n);\n\n// Create task\nfinal task = await projectApi.createTask(\n  workspaceId,\n  projectId,\n  CreateTaskDto(\n    title: 'New Task',\n    description: 'Task description',\n    status: 'todo',\n    priority: 'medium',\n  ),\n);\n\n// Get project tasks\nfinal tasks = await projectApi.getProjectTasks(\n  workspaceId,\n  projectId,\n  status: 'in_progress',\n);\n```\n\n### 3. Calendar Operations\n\n```dart\nfinal calendarApi = CalendarApiService();\n\n// Create event\nfinal event = await calendarApi.createEvent(\n  workspaceId,\n  CreateEventDto(\n    title: 'Team Meeting',\n    description: 'Weekly team sync',\n    startTime: DateTime.now(),\n    endTime: DateTime.now().add(Duration(hours: 1)),\n    attendeeEmails: ['user1@example.com', 'user2@example.com'],\n  ),\n);\n\n// Get events for date range\nfinal events = await calendarApi.getEvents(\n  workspaceId,\n  startDate: '2025-01-01',\n  endDate: '2025-01-31',\n);\n```\n\n### 4. File Operations\n\n```dart\nfinal fileApi = FileApiService();\n\n// Upload file\nfinal file = File('/path/to/file.pdf');\nfinal uploadedFile = await fileApi.uploadFile(\n  workspaceId,\n  file,\n  UploadFileDto(\n    description: 'Important document',\n    tags: ['work', 'important'],\n  ),\n);\n\n// Download file\nawait fileApi.downloadFile(\n  workspaceId,\n  fileId,\n  '/path/to/save/file.pdf',\n);\n\n// Search files\nfinal searchResults = await fileApi.searchFiles(\n  workspaceId,\n  'meeting notes',\n  mimeType: 'application/pdf',\n);\n```\n\n### 5. Chat Operations\n\n```dart\nfinal chatApi = ChatApiService();\n\n// Create channel\nfinal channel = await chatApi.createChannel(\n  workspaceId,\n  CreateChannelDto(\n    name: 'general',\n    description: 'General discussion',\n    type: 'public',\n  ),\n);\n\n// Send message\nfinal message = await chatApi.sendChannelMessage(\n  workspaceId,\n  channelId,\n  SendMessageDto(\n    content: 'Hello team!',\n    type: 'text',\n  ),\n);\n```\n\n### 6. Notification Management\n\n```dart\nfinal notificationApi = NotificationApiService();\n\n// Send notification\nawait notificationApi.sendNotification(\n  CreateNotificationDto(\n    title: 'Task Completed',\n    body: 'Your task has been completed',\n    type: 'success',\n    category: 'task',\n    priority: 'medium',\n  ),\n);\n\n// Get notifications\nfinal notifications = await notificationApi.getNotifications(\n  NotificationQueryDto(\n    isRead: false,\n    limit: 50,\n  ),\n);\n\n// Update preferences\nawait notificationApi.updateNotificationPreferences(\n  UpdatePreferencesDto(\n    global: NotificationGlobalSettings(\n      push: true,\n      email: false,\n      inApp: true,\n    ),\n  ),\n);\n```\n\n## Error Handling\n\nAll API services return `ApiResponse<T>` objects with consistent error handling:\n\n```dart\nfinal response = await projectApi.getAllProjects(workspaceId);\n\nif (response.success && response.data != null) {\n  // Success case\n  final projects = response.data!;\n  debugPrint('Loaded ${projects.length} projects');\n} else {\n  // Error case\n  debugPrint('Error: ${response.message}');\n  debugPrint('Status: ${response.statusCode}');\n  \n  // Handle specific error codes\n  switch (response.statusCode) {\n    case 401:\n      // Handle unauthorized\n      break;\n    case 403:\n      // Handle forbidden\n      break;\n    case 404:\n      // Handle not found\n      break;\n    default:\n      // Handle other errors\n  }\n}\n```\n\n## Configuration\n\n### Base URL\nThe base URL is set in `/lib/api/base_api_client.dart`:\n\n```dart\nstatic const String baseUrl = 'http://localhost:4000';\n```\n\nUpdate this for different environments (development, staging, production).\n\n### Authentication\nJWT tokens are automatically included from `AuthService.instance.currentSession`. No manual token management needed.\n\n### Logging\nRequest/response logging is enabled in debug mode. Disable by removing the `LogInterceptor` from `base_api_client.dart`.\n\n## Best Practices\n\n### 1. Service Singleton Pattern\n\n```dart\nclass MyService {\n  static MyService? _instance;\n  static MyService get instance => _instance ??= MyService._internal();\n  MyService._internal();\n  \n  final ProjectApiService _projectApi = ProjectApiService();\n  // ... service methods\n}\n```\n\n### 2. Error Handling\n\nAlways handle both success and error cases:\n\n```dart\nFuture<List<Project>> getProjects() async {\n  try {\n    final response = await _projectApi.getAllProjects(workspaceId);\n    \n    if (response.success && response.data != null) {\n      return response.data!;\n    } else {\n      // Log error but don't crash the app\n      debugPrint('API Error: ${response.message}');\n      return []; // Return empty list instead of null\n    }\n  } catch (e) {\n    debugPrint('Network Error: $e');\n    return []; // Graceful fallback\n  }\n}\n```\n\n### 3. Caching\n\nImplement caching for frequently accessed data:\n\n```dart\nclass ProjectService {\n  List<Project>? _cachedProjects;\n  DateTime? _lastFetch;\n  \n  Future<List<Project>> getProjects({bool forceRefresh = false}) async {\n    // Return cache if fresh (< 5 minutes)\n    if (!forceRefresh && \n        _cachedProjects != null && \n        _lastFetch != null &&\n        DateTime.now().difference(_lastFetch!).inMinutes < 5) {\n      return _cachedProjects!;\n    }\n    \n    // Fetch from API\n    final response = await _projectApi.getAllProjects(workspaceId);\n    if (response.success && response.data != null) {\n      _cachedProjects = response.data!;\n      _lastFetch = DateTime.now();\n      return _cachedProjects!;\n    }\n    \n    return _cachedProjects ?? [];\n  }\n}\n```\n\n## Migration Checklist\n\n- [ ] Run `flutter pub get` to install Dio\n- [ ] Update existing services to use new API clients\n- [ ] Replace direct HTTP calls with API service calls\n- [ ] Update error handling to use ApiResponse pattern\n- [ ] Test all API operations with the new client\n- [ ] Update base URL for your environment\n- [ ] Verify JWT token authentication works\n- [ ] Test error scenarios (network issues, 401, 404, etc.)\n- [ ] Update any UI code that depends on service responses\n- [ ] Remove old HTTP utility code if no longer needed\n\n## Example: Complete Service Migration\n\nSee `/lib/services/project_service_updated.dart` for a complete example of migrating from direct HTTP calls to the new API client system.\n\n## Support\n\nIf you encounter issues during migration:\n1.
