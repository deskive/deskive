## Deskive란 무엇인가요?

Deskive는 실시간 커뮤니케이션, 프로젝트 관리 및 생산성 도구를 하나로 결합한 **셀프 호스팅 가능한 워크스페이스 협업 플랫폼**입니다. 데이터를 완전히 제어하고자 하는 팀을 위해 구축된 Deskive는 Slack + Notion + Zoom + Asana의 기능을 단일 오픈소스 애플리케이션으로 제공합니다.

화상 통화를 위해 유료 플랜이 필요한 Slack이나 실시간 채팅이 부족한 Notion과 달리, Deskive는 효과적인 협업을 위해 필요한 모든 것(채팅, 화상 통화, 프로젝트 보드, 파일 공유, AI 지원)을 벤더 종속이나 독점 라이선스 없이 제공합니다.

### 작동 방식

1. **워크스페이스 생성** -- 채널, 프로젝트 및 사용자 정의 역할로 팀 워크스페이스를 설정합니다
2. **실시간 커뮤니케이션** -- 스레드, 반응, 멘션, GIF 및 HD 화상 통화를 통해 채팅합니다
3. **프로젝트 관리** -- Kanban 보드, 스프린트, 작업 의존성 및 시간 추적으로 작업을 구성합니다
4. **문서 공동 작업** -- 버전 관리 및 디지털 서명이 있는 노트, 화이트보드, 파일을 공유합니다
5. **AI로 자동화** -- AutoPilot이 일정 관리, 회의 요약 및 일일 브리핑을 처리하도록 합니다

### 주요 기능

- **💬 실시간 커뮤니케이션** -- 채널, 다이렉트 메시지, 스레드, 반응, 멘션 및 GIF 지원
- **📹 HD 화상 회의** -- LiveKit을 통한 화면 공유, 녹화 및 전사 기능이 있는 내장 화상 통화
- **📋 프로젝트 관리** -- Kanban 보드, 스프린트, 마일스톤, 작업 의존성 및 시간 추적
- **📁 파일 관리** -- 버전 관리, 공유 및 Google Drive 통합이 있는 클라우드 스토리지
- **📝 협업 노트** -- 실시간 협업 및 템플릿이 있는 블록 기반 에디터
- **📅 캘린더 및 일정 관리** -- 이벤트 관리, 반복 이벤트, 회의실 및 가용성 추적
- **🎨 화이트보드** -- 브레인스토밍 및 계획을 위한 시각적 협업 워크스페이스
- **🤖 AI 어시스턴트** -- 일정 관리, 회의 인텔리전스 및 문서 분석을 위한 AutoPilot
- **📊 양식 및 분석** -- 응답 추적 및 워크스페이스 메트릭이 있는 사용자 정의 양식 빌더
- **✅ 승인 워크플로** -- 문서 및 프로세스를 위한 내장 승인 시스템
- **💰 예산 추적** -- 비용 관리, 청구 요금 및 예산 모니터링
- **🔗 통합** -- Slack, Google Drive, GitHub, Dropbox 등과 연결
- **🔍 시맨틱 검색** -- 모든 콘텐츠 유형에 걸친 AI 기반 검색
- **🌍 국제화** -- 다국어 지원 (영어, 일본어, 확장 가능)

## 우리가 해결하는 문제

### 협업 도구 분산의 딜레마

현대 팀은 여러 구독을 관리합니다: 채팅용 Slack($8.75/사용자/월), 비디오용 Zoom($15.99/사용자/월), 프로젝트용 Asana($10.99/사용자/월), 문서용 Notion($10/사용자/월). 이는 분산된 워크플로, 데이터 사일로, 여러 벤더로 인한 보안 위험 및 팀 규모에 따라 선형적으로 증가하는 비용을 초래합니다.

**우리가 해결하는 일반적인 문제점:**

- ❌ **도구 분산** -- 매일 5개 이상의 도구를 전환하면 집중력과 생산성이 방해받습니다
- ❌ **비용 증가** -- 기본 협업을 위한 SaaS 구독 비용이 사용자당 월 $50 이상으로 증가합니다
- ❌ **데이터 종속** -- 제한된 내보내기 옵션으로 타인의 서버에 데이터가 저장됩니다
- ❌ **개인정보 보호 문제** -- 민감한 비즈니스 데이터가 여러 타사 벤더와 공유됩니다
- ❌ **통합 복잡성** -- 각 도구마다 별도의 API 통합 및 인증이 필요합니다
- ❌ **기능 격차** -- 포괄적인 협업 기능을 제공하는 단일 플랫폼이 없습니다

### Deskive의 솔루션

✅ **올인원 플랫폼** -- 채팅, 비디오, 프로젝트, 파일, 캘린더, 노트 및 AI가 하나의 애플리케이션에

✅ **셀프 호스팅 및 오픈소스** -- GNU AGPL 3.0 라이선스로 완전한 데이터 소유권

✅ **사용자당 비용 제로** -- 팀 규모에 관계없이 하나의 인프라 비용

✅ **깊은 통합** -- 모든 기능이 컨텍스트와 데이터를 원활하게 공유

✅ **엔터프라이즈급** -- 디지털 서명, 승인 워크플로, 감사 로그 및 SSO 지원

### Deskive를 특별하게 만드는 것은?

1. **진정한 통합 플랫폼** -- 모든 기능이 동일한 데이터 모델을 공유하여 별도 도구로는 불가능한 깊은 통합을 가능하게 합니다
2. **타협 없는 셀프 호스팅** -- 화상 통화 및 AI를 포함한 SaaS 대안과 완전한 기능 동등성
3. **현대적인 기술 스택** -- 유지보수성과 성능을 위해 React 19, NestJS 11 및 TypeScript로 구축
4. **AI 네이티브 설계** -- 핵심 플랫폼에 내장된 벡터 검색, 대화 메모리 및 AutoPilot 에이전트
5. **비용 효율적인 확장** -- 사용자당 요금제인 SaaS와 달리 하나의 인프라 비용으로 무제한 사용자 지원

## 빠른 시작

### Docker (권장)

프로젝트 루트에서 다음 명령을 실행하세요:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# 구성으로 .env를 편집하세요 (데이터베이스 자격 증명, API 키 등)
docker compose up -d
```

그게 다입니다! `http://localhost:5175`에서 앱에 액세스하고 `http://localhost:3000`에서 API에 액세스하세요.

### 수동 설정

**필수 조건:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# 클론
git clone https://github.com/deskive/deskive.git
cd deskive

# 백엔드
cd backend
cp .env.example .env    # 구성으로 .env를 편집하세요
npm install
npm run migrate         # 데이터베이스 마이그레이션 실행
npm run start:dev

# 프론트엔드 (새 터미널에서)
cd frontend
cp .env.example .env
npm install
npm run dev
```

프론트엔드: `http://localhost:5175` | 백엔드: `http://localhost:3000`

### 원 커맨드 시작

개발 환경의 경우:

```bash
./start.sh
```

**Frontend** (`/frontend`) -- Vite, TypeScript, Tailwind CSS, Radix UI 컴포넌트, 상태 관리를 위한 Zustand, 데이터 페칭을 위한 React Query를 사용한 React 19

**Backend** (`/backend`) -- TypeScript, 원시 SQL 쿼리를 사용한 PostgreSQL, 캐싱 및 실시간 기능을 위한 Redis, WebSocket 연결을 위한 Socket.io를 사용한 NestJS 11

**AI 및 검색** -- 벡터 임베딩을 위한 Qdrant, GPT-4o-mini 및 Whisper 전사를 위한 OpenAI

**비디오** -- HD 화상 통화, 화면 공유, 녹화 및 실시간 전사를 위한 LiveKit

## 플러그형 프로바이더

모든 백엔드 서비스는 단일 환경 변수로 교체할 수 있습니다. 기본 설정에서는 클라우드 자격 증명 없이 Deskive를 실행할 수 있으며, 준비가 되면 관리형 프로바이더로 전환하세요.

| 도메인 | 환경 변수 | 제공되는 프로바이더 |
|---|---|---|
| **스토리지** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (기본값), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **이메일** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **푸시** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **검색** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (기본값, 추가 인프라 불필요), `meilisearch`, `typesense`, `none` |
| **인증 / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (비밀번호 없는, JWT 기반) |
| **비디오** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **AI** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (로컬) |

- **키워드 검색과 시맨틱 검색이 공존합니다.** `SearchProviderService`는 트라이그램/패싯 키워드 검색을 처리하고, `SearchService`는 계속해서 Qdrant 벡터/시맨틱 검색을 담당합니다.
- **선택적 SDK는 지연 로드됩니다** — `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk`, `agora-token`은 `optionalDependencies`이므로 `local-fs` / `smtp` / `webpush` / `pg-trgm`을 선택해도 설치 비용이 전혀 들지 않습니다.
- **스모크 테스트**가 각 어댑터와 함께 제공됩니다 (`backend/scripts/smoke-test-*-providers.ts`) — 스토리지 / 이메일 / 푸시 / 검색 / 인증에서 각각 27 / 45 / 61 / 55 / 37개의 어설션이 있으며 모두 통과합니다.
- **전체 문서:** 프로바이더별 환경 변수와 설정은 `backend/docs/providers/`를 참조하세요.

## 🚀 왜 Deskive에 기여해야 하나요?

Deskive는 단순한 오픈소스 프로젝트가 아닙니다 -- 현대적인 개발 관행을 마스터하면서 팀 협업의 미래를 구축할 수 있는 기회입니다.

### 얻을 수 있는 것

**📚 현대적인 기술 스택 학습**
- **React 19** -- 동시성 기능 및 서버 컴포넌트가 있는 최신 React
- **NestJS 11** -- 의존성 주입이 있는 엔터프라이즈급 Node.js 프레임워크
- **전체적인 TypeScript** -- 강력한 타이핑, 더 나은 IDE 지원, 더 적은 버그
- **PostgreSQL + 원시 SQL** -- ORM 없이 데이터베이스 설계
- **실시간 시스템** -- WebSocket용 Socket.io, pub/sub용 Redis
- **AI 통합** -- OpenAI 임베딩, Qdrant를 사용한 벡터 검색

**💼 포트폴리오 구축**
- 전 세계 팀이 사용하는 **프로덕션 준비** 플랫폼에 기여
- GitHub 프로필에 표시되는 기능 작업
- 기여자 명예의 전당에서 인정받기
- **협업 플랫폼** 및 **실시간 시스템** 전문 지식 구축 -- 2026년에 매우 가치 있는 기술

**🤝 성장하는 커뮤니티 참여**
- 전 세계 개발자와 연결
- 경험 많은 유지관리자로부터 코드 리뷰 받기
- 소프트웨어 아키텍처의 모범 사례 학습
- 기술 토론 및 설계 결정 참여

**🎯 실질적인 영향 만들기**
- 여러분의 코드가 팀이 비싼 SaaS 구독에서 벗어나도록 도와줍니다
- 프로덕션 환경에서 사용되는 기능 확인
- 오픈소스 협업 도구의 방향에 영향을 미침

**⚡ 빠른 온보딩**
- Docker Compose로 **5분 이내**에 실행
- 명확한 아키텍처로 잘 문서화된 코드베이스
- PR에 48시간 이내에 응답하는 친절한 유지관리자
- 초보자를 위한 "Good first issue" 레이블

## 🎯 빠른 기여 가이드

**10분 이내**에 기여를 시작하세요:

### 1단계: 환경 설정

```bash
# GitHub에서 리포지토리를 포크한 다음 포크를 클론하세요
git clone https://github.com/YOUR_USERNAME/deskive.git
cd deskive

# Docker로 시작 (가장 쉬운 방법)
cp .env.docker .env
docker compose up -d

# 앱에 액세스
# 프론트엔드: http://localhost:5175
# 백엔드 API: http://localhost:3000
```

**그게 다입니다!** 이제 로컬에서 Deskive를 실행하고 있습니다.

### 3단계: 변경 사항 만들기

```bash
# 새 브랜치 생성
git checkout -b feature/your-feature-name

# 변경 사항 만들기
# - 백엔드 코드: /backend/src/modules
# - 프론트엔드 코드: /frontend/src
# - 데이터베이스 마이그레이션: /backend/migrations

# 변경 사항 테스트
npm test

# 명확한 메시지로 커밋
git commit -m "feat: add new integration for XYZ"
```

### 4단계: Pull Request 제출

```bash
# 포크에 푸시
git push origin feature/your-feature-name

# GitHub에서 PR 열기
# - 무엇을 변경했고 왜 변경했는지 설명하세요
# - 관련 이슈에 링크하세요
# - UI 변경인 경우 스크린샷 추가
```

**다음에 무슨 일이 일어나나요?**
- ✅ PR에서 자동화된 테스트 실행
- 👀 유지관리자가 코드 검토 (보통 48시간 이내)
- 💬 변경 사항 또는 개선 사항을 제안할 수 있습니다
