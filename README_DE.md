## Was ist Deskive?

Deskive ist eine **selbst-hostbare Plattform für Workspace-Zusammenarbeit**, die Echtzeit-Kommunikation, Projektmanagement und Produktivitätstools vereint. Entwickelt für Teams, die vollständige Kontrolle über ihre Daten wollen, bietet Deskive Ihnen die Funktionalität von Slack + Notion + Zoom + Asana in einer einzigen Open-Source-Anwendung.

Im Gegensatz zu Slack, das kostenpflichtige Tarife für Videoanrufe erfordert, oder Notion, dem Echtzeit-Chat fehlt, bietet Ihnen Deskive alles, was Sie für eine effektive Zusammenarbeit benötigen -- Chat, Videoanrufe, Projektboards, Dateifreigabe, KI-Assistenz -- ohne Vendor Lock-in oder proprietäre Lizenzierung.

### Wie es funktioniert

1. **Erstellen Sie Ihren Workspace** -- Richten Sie Team-Workspaces mit Kanälen, Projekten und benutzerdefinierten Rollen ein
2. **Kommunizieren Sie in Echtzeit** -- Chatten Sie mit Threads, Reaktionen, Erwähnungen, GIFs und HD-Videoanrufen
3. **Projekte verwalten** -- Organisieren Sie Arbeit mit Kanban-Boards, Sprints, Aufgabenabhängigkeiten und Zeiterfassung
4. **Gemeinsam an Dokumenten arbeiten** -- Teilen Sie Notizen, Whiteboards, Dateien mit Versionskontrolle und digitalen Signaturen
5. **Mit KI automatisieren** -- Lassen Sie AutoPilot Terminplanung, Meeting-Zusammenfassungen und tägliche Briefings übernehmen

### Hauptfunktionen

- **💬 Echtzeit-Kommunikation** -- Kanäle, Direktnachrichten, Threads, Reaktionen, Erwähnungen und GIF-Unterstützung
- **📹 HD-Videokonferenzen** -- Integrierte Videoanrufe mit Bildschirmfreigabe, Aufzeichnung und Transkription über LiveKit
- **📋 Projektmanagement** -- Kanban-Boards, Sprints, Meilensteine, Aufgabenabhängigkeiten und Zeiterfassung
- **📁 Dateiverwaltung** -- Cloud-Speicher mit Versionierung, Freigabe und Google Drive Integration
- **📝 Gemeinsame Notizen** -- Blockbasierter Editor mit Echtzeit-Zusammenarbeit und Vorlagen
- **📅 Kalender & Terminplanung** -- Veranstaltungsverwaltung, wiederkehrende Termine, Besprechungsräume und Verfügbarkeitsverfolgung
- **🎨 Whiteboard** -- Visueller Kollaborations-Workspace für Brainstorming und Planung
- **🤖 KI-Assistent** -- AutoPilot für Terminplanung, Meeting-Intelligence und Dokumentenanalyse
- **📊 Formulare & Analytics** -- Benutzerdefinierter Formular-Builder mit Antwortverfolgung und Workspace-Metriken
- **✅ Genehmigungs-Workflows** -- Integriertes Genehmigungssystem für Dokumente und Prozesse
- **💰 Budget-Tracking** -- Kostenverwaltung, Abrechnungssätze und Budget-Überwachung
- **🔗 Integrationen** -- Verbindung mit Slack, Google Drive, GitHub, Dropbox und mehr
- **🔍 Semantische Suche** -- KI-gestützte Suche über alle Inhaltstypen
- **🌍 Internationalisierung** -- Mehrsprachige Unterstützung (Englisch, Japanisch, erweiterbar)

## Welches Problem wir lösen

### Das Dilemma der Fragmentierung von Kollaborations-Tools

Moderne Teams jonglieren mit mehreren Abonnements: Slack für Chat ($8,75/Nutzer/Monat), Zoom für Video ($15,99/Nutzer/Monat), Asana für Projekte ($10,99/Nutzer/Monat), Notion für Dokumente ($10/Nutzer/Monat). Dies führt zu fragmentierten Workflows, Datensilos, Sicherheitsrisiken durch mehrere Anbieter und Kosten, die linear mit der Teamgröße skalieren.

**Häufige Schmerzpunkte, die wir adressieren:**

- ❌ **Tool-Fragmentierung** -- Täglicher Wechsel zwischen 5+ Tools stört Fokus und Produktivität
- ❌ **Steigende Kosten** -- SaaS-Abonnements summieren sich auf $50+/Nutzer/Monat für grundlegende Zusammenarbeit
- ❌ **Daten-Lock-In** -- Ihre Daten befinden sich auf fremden Servern mit begrenzten Export-Optionen
- ❌ **Datenschutzbedenken** -- Sensible Geschäftsdaten werden mit mehreren Drittanbietern geteilt
- ❌ **Integrations-Komplexität** -- Jedes Tool erfordert separate API-Integrationen und Authentifizierung
- ❌ **Feature-Lücken** -- Keine einzelne Plattform bietet umfassende Kollaborationsfunktionen

### Deskives Lösung

✅ **All-in-One-Plattform** -- Chat, Video, Projekte, Dateien, Kalender, Notizen und KI in einer Anwendung

✅ **Selbst-gehostet & Open Source** -- Vollständiger Datenbesitz mit GNU AGPL 3.0 Lizenz

✅ **Null Kosten pro Nutzer** -- Eine Infrastrukturkosten unabhängig von der Teamgröße

✅ **Tiefe Integration** -- Alle Funktionen teilen nahtlos Kontext und Daten

✅ **Enterprise-Ready** -- Digitale Signaturen, Genehmigungs-Workflows, Audit-Logs und SSO-Unterstützung

### Was macht Deskive einzigartig?

1. **Wirklich vereinheitlichte Plattform** -- Alle Funktionen teilen dasselbe Datenmodell und ermöglichen tiefe Integration, die mit separaten Tools unmöglich ist
2. **Self-Hosting ohne Kompromisse** -- Vollständige Feature-Parität mit SaaS-Alternativen, einschließlich Videoanrufen und KI
3. **Moderner Tech-Stack** -- Entwickelt mit React 19, NestJS 11 und TypeScript für Wartbarkeit und Performance
4. **KI-natives Design** -- Vektorsuche, Conversation Memory und AutoPilot-Agent in die Kernplattform integriert
5. **Kosteneffektive Skalierung** -- Eine Infrastrukturkosten bedient unbegrenzt viele Nutzer, im Gegensatz zu Preisen pro Sitzplatz bei SaaS

## Schnellstart

### Docker (Empfohlen)

Führen Sie diese Befehle vom Projektstamm aus:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# Bearbeiten Sie .env mit Ihrer Konfiguration (Datenbank-Credentials, API-Schlüssel, etc.)
docker compose up -d
```

Das war's! Greifen Sie auf die App unter `http://localhost:5175` und die API unter `http://localhost:3000` zu.

### Manuelle Einrichtung

**Voraussetzungen:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# Klonen
git clone https://github.com/deskive/deskive.git
cd deskive

# Backend
cd backend
cp .env.example .env    # Bearbeiten Sie .env mit Ihrer Konfiguration
npm install
npm run migrate         # Datenbank-Migrationen ausführen
npm run start:dev

# Frontend (in einem neuen Terminal)
cd frontend
cp .env.example .env
npm install
npm run dev
```

Frontend: `http://localhost:5175` | Backend: `http://localhost:3000`

### Ein-Befehl-Start

Für Entwicklungsumgebungen:

```bash
./start.sh
```

```

└────────────────────────┬────────────────────────────────────┘
                         │ REST API + Socket.io
┌────────────────────────┴────────────────────────────────────┐
│                    Backend (NestJS 11)                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│  │   Auth   │  │   Chat   │  │  Tasks   │  │    AI    │     │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘     │
│         40+ Module mit TypeScript + Raw SQL                 │
└────────┬─────────────┬─────────────┬─────────────┬──────────┘
         │             │             │             │
    ┌────┴────┐   ┌────┴────┐   ┌────┴────┐   ┌────┴────┐
    │Postgres │   │  Redis  │   │ Qdrant  │   │LiveKit  │
    │(Storage)│   │(Cache)  │   │(Vector) │   │(Video)  │
    └─────────┘   └─────────┘   └─────────┘   └─────────┘
```

**Frontend** (`/frontend`) -- React 19 mit Vite, TypeScript, Tailwind CSS, Radix UI Komponenten, Zustand für State Management, React Query für Data Fetching

**Backend** (`/backend`) -- NestJS 11 mit TypeScript, PostgreSQL mit rohen SQL-Abfragen, Redis für Caching und Echtzeit-Features, Socket.io für WebSocket-Verbindungen

**KI & Suche** -- Qdrant für Vektor-Embeddings, OpenAI für GPT-4o-mini und Whisper-Transkription

**Video** -- LiveKit für HD-Videoanrufe, Bildschirmfreigabe, Aufzeichnung und Echtzeit-Transkription

## Austauschbare Provider

Jeder zugrunde liegende Dienst ist über eine einzelne Umgebungsvariable austauschbar. Die Standardwerte halten Deskive ohne Cloud-Zugangsdaten lauffähig; wechseln Sie zu einem Managed Provider, wenn Sie bereit sind.

| Domäne | Umgebungsvariable | Mitgelieferte Provider |
|---|---|---|
| **Speicher** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (Standard), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **E-Mail** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **Push** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **Suche** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (Standard, keine zusätzliche Infrastruktur), `meilisearch`, `typesense`, `none` |
| **Auth / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (passwortlos, JWT-basiert) |
| **Video** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **KI** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (lokal) |

- **Keyword- und semantische Suche koexistieren.** `SearchProviderService` übernimmt Trigramm-/Facettensuche nach Keywords; `SearchService` ist weiterhin für die Qdrant-Vektor-/Semantiksuche zuständig.
- **Optionale SDKs werden lazy geladen** — `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk` und `agora-token` sind `optionalDependencies`, sodass die Wahl von `local-fs` / `smtp` / `webpush` / `pg-trgm` keinerlei Installationskosten verursacht.
- **Smoke-Tests** werden mit jedem Adapter ausgeliefert (`backend/scripts/smoke-test-*-providers.ts`) — 27 / 45 / 61 / 55 / 37 Assertions für Storage / E-Mail / Push / Suche / Auth, alle erfolgreich.
- **Vollständige Dokumentation:** Siehe `backend/docs/providers/` für Provider-spezifische Umgebungsvariablen und Einrichtung.

- Docker Compose bringt Sie in **unter 5 Minuten** zum Laufen
- Gut dokumentierte Codebasis mit klarer Architektur
- Freundliche Maintainer, die innerhalb von 48 Stunden auf PRs reagieren
- "Good first issue"-Labels für Neueinsteiger

## 🎯 Schneller Beitrags-Leitfaden

Beginnen Sie mit Beiträgen in **unter 10 Minuten**:

### Schritt 1: Richten Sie Ihre Umgebung ein

```bash
# Forken Sie das Repository auf GitHub, dann klonen Sie Ihren Fork
git clone https://github.com/IHR_BENUTZERNAME/deskive.git
cd deskive

# Starten Sie mit Docker (einfachster Weg)
cp .env.docker .env
docker compose up -d

# Greifen Sie auf die App zu
# Frontend: http://localhost:5175
# Backend API: http://localhost:3000
```

**Das war's!** Sie führen Deskive lokal aus.

### Schritt 2: Finden Sie etwas zum Bearbeiten

Wählen Sie basierend auf Ihrem Erfahrungslevel:

**🟢 Anfängerfreundlich**
- 📝 [Tippfehler beheben oder Dokumentation verbessern](https://github.com/deskive/deskive/labels/documentation)
- 🌍 [Übersetzungen hinzufügen](https://github.com/deskive/deskive/labels/i18n) -- Wir unterstützen Englisch und Japanisch
- 🐛 [Einfache Bugs beheben](https://github.com/deskive/deskive/labels/good%20first%20issue)
- ✨ [UI/UX verbessern](https://github.com/deskive/deskive/labels/ui%2Fux)

**🟡 Fortgeschritten**
- 🔗 Neue Integrationen hinzufügen -- Siehe unseren [Integrationsleitfaden](backend/README.md#integrations)
- 🧪 [Tests schreiben](https://github.com/deskive/deskive/labels/tests)
- 🚀 [Performance-Verbesserungen](https://github.com/deskive/deskive/labels/performance)
- 📱 [Mobile-Responsiveness](https://github.com/deskive/deskive/labels/mobile)

**🔴 Erweitert**
- 🤖 [KI-Features](https://github.com/deskive/deskive/labels/ai) -- AutoPilot-Verbesserungen, neue KI-Fähigkeiten
- ⚙️ [Core-Engine-Verbesserungen](https://github.com/deskive/deskive/labels/core)
- 🏗️ [Architektur-Verbesserungen](https://github.com/deskive/deskive/labels/architecture)
- 🔐 [Sicherheits-Features](https://github.com/deskive/deskive/labels/security)

### Schritt 3: Nehmen Sie Ihre Änderungen vor

```bash
# Erstellen Sie einen neuen Branch
git checkout -b feature/ihr-feature-name

# Nehmen Sie Ihre Änderungen vor
# - Backend-Code: /backend/src/modules
# - Frontend-Code: /frontend/src
# - Datenbank-Migrationen: /backend/migrations

# Testen Sie Ihre Änderungen
npm test

# Commit mit klarer Nachricht
git commit -m "feat: neue Integration für XYZ hinzufügen"
```

### Schritt 4: Reichen Sie Ihren Pull Request ein

```bash
# Pushen Sie zu Ihrem Fork
git push origin feature/ihr-feature-name

# Öffnen Sie einen PR auf GitHub
# - Beschreiben Sie, was Sie geändert haben und warum
# - Verlinken Sie zu verwandten Issues
# - Fügen Sie Screenshots hinzu, wenn es eine UI-Änderung ist
```

**Was passiert als nächstes?**
- ✅ Automatisierte Tests laufen auf Ihrem PR
- 👀 Ein Maintainer überprüft Ihren Code (normalerweise innerhalb von 48 Stunden)
- 💬 Wir könnten Änderungen oder Verbesserungen vorschlagen
- 🎉 Nach Genehmigung wird Ihr Code gemerged!

### Beitrags-Tipps

✨ **Beginnen Sie klein** -- Ihr erster PR muss kein riesiges Feature sein

📖 **Lesen Sie den Code** -- Durchsuchen Sie bestehende Module in `backend/src/modules` zur Referenz

❓ **Stellen Sie Fragen** -- Öffnen Sie eine [GitHub-Diskussion](https://github.com/deskive/deskive/discussions), wenn Sie nicht weiterkommen

🧪 **Schreiben Sie Tests** -- PRs mit Tests werden schneller gemerged

📝 **Dokumentieren Sie Ihren Code** -- Fügen Sie Kommentare für komplexe Logik hinzu
