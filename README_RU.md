## Что такое Deskive?

Deskive — это **платформа для совместной работы с возможностью самостоятельного хостинга**, которая объединяет общение в реальном времени, управление проектами и инструменты повышения продуктивности. Создана для команд, которым нужен полный контроль над своими данными. Deskive предоставляет функциональность Slack + Notion + Zoom + Asana в одном приложении с открытым исходным кодом.

В отличие от Slack, требующего платных тарифов для видеозвонков, или Notion, в котором отсутствует чат в реальном времени, Deskive предоставляет всё необходимое для эффективного сотрудничества -- чат, видеозвонки, проектные доски, обмен файлами, помощь AI -- без привязки к поставщику или проприетарных лицензий.

### Как это работает

1. **Создайте своё рабочее пространство** -- Настройте командные рабочие пространства с каналами, проектами и пользовательскими ролями
2. **Общайтесь в реальном времени** -- Чат с ветками, реакциями, упоминаниями, GIF и видеозвонками в HD
3. **Управляйте проектами** -- Организуйте работу с досками Kanban, спринтами, зависимостями задач и учётом времени
4. **Работайте над документами совместно** -- Делитесь заметками, досками, файлами с контролем версий и цифровыми подписями
5. **Автоматизируйте с помощью AI** -- Позвольте AutoPilot управлять планированием, резюме встреч и ежедневными сводками

## Какую проблему мы решаем

### Дилемма фрагментации инструментов для совместной работы

Современные команды жонглируют несколькими подписками: Slack для чата ($8.75/пользователь/мес.), Zoom для видео ($15.99/пользователь/мес.), Asana для проектов ($10.99/пользователь/мес.), Notion для документов ($10/пользователь/мес.). Это создаёт фрагментированные рабочие процессы, изолированные данные, риски безопасности от множества поставщиков и затраты, которые линейно растут с размером команды.

**Распространённые проблемы, которые мы решаем:**

- ❌ **Фрагментация инструментов** -- Переключение между 5+ инструментами ежедневно нарушает концентрацию и продуктивность
- ❌ **Растущие затраты** -- Подписки SaaS в сумме превышают $50/пользователь/месяц для базового сотрудничества
- ❌ **Привязка к данным** -- Ваши данные хранятся на чужих серверах с ограниченными возможностями экспорта
- ❌ **Проблемы конфиденциальности** -- Конфиденциальные бизнес-данные передаются множеству сторонних поставщиков
- ❌ **Сложность интеграции** -- Каждый инструмент требует отдельных API-интеграций и аутентификации
- ❌ **Пробелы в функциональности** -- Ни одна платформа не предлагает комплексные функции для совместной работы

### Решение Deskive

✅ **Платформа всё-в-одном** -- Чат, видео, проекты, файлы, календарь, заметки и AI в одном приложении

✅ **Самостоятельный хостинг и открытый исходный код** -- Полное владение данными с лицензией GNU AGPL 3.0

✅ **Нулевые затраты на пользователя** -- Одна стоимость инфраструктуры независимо от размера команды

✅ **Глубокая интеграция** -- Все функции обмениваются контекстом и данными без проблем

✅ **Готовность для предприятий** -- Цифровые подписи, рабочие процессы утверждения, журналы аудита и поддержка SSO

### Что делает Deskive уникальным?

1. **Действительно единая платформа** -- Все функции используют одну модель данных, обеспечивая глубокую интеграцию, невозможную с отдельными инструментами
2. **Самостоятельный хостинг без компромиссов** -- Полный паритет функций с SaaS-альтернативами, включая видеозвонки и AI
3. **Современный технологический стек** -- Создан на React 19, NestJS 11 и TypeScript для удобства сопровождения и производительности
4. **AI-нативный дизайн** -- Векторный поиск, память разговоров и агент AutoPilot встроены в ядро платформы
5. **Экономически эффективное масштабирование** -- Одна стоимость инфраструктуры обслуживает неограниченное количество пользователей, в отличие от посадочных мест SaaS

## Быстрый старт

### Docker (рекомендуется)

Выполните эти команды из корневой директории проекта:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# Отредактируйте .env с вашей конфигурацией (учётные данные базы данных, API-ключи и т.д.)
docker compose up -d
```

Вот и всё! Доступ к приложению по адресу `http://localhost:5175` и к API по адресу `http://localhost:3000`.

### Ручная установка

**Требования:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# Клонирование
git clone https://github.com/deskive/deskive.git
cd deskive

# Backend
cd backend
cp .env.example .env    # Отредактируйте .env с вашей конфигурацией
npm install
npm run migrate         # Запустите миграции базы данных
npm run start:dev

# Frontend (в новом терминале)
cd frontend
cp .env.example .env
npm install
npm run dev
```

Frontend: `http://localhost:5175` | Backend: `http://localhost:3000`

### Запуск одной командой

Для сред разработки:

```bash
./start.sh
```

## Подключаемые провайдеры

Каждый внутренний сервис заменяется одной переменной окружения. Значения по умолчанию позволяют запускать Deskive без каких-либо облачных учётных данных; переключайтесь на управляемого провайдера, когда будете готовы.

| Домен | Переменная окружения | Входящие провайдеры |
|---|---|---|
| **Хранилище** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (по умолчанию), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **Почта** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **Push** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **Поиск** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (по умолчанию, без дополнительной инфраструктуры), `meilisearch`, `typesense`, `none` |
| **Аутентификация / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (без пароля, на основе JWT) |
| **Видео** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **ИИ** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (локально) |

- **Ключевой и семантический поиск сосуществуют.** `SearchProviderService` обрабатывает триграммный/фасетный поиск по ключевым словам; `SearchService` по-прежнему отвечает за векторный/семантический поиск Qdrant.
- **Опциональные SDK загружаются лениво** — `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk` и `agora-token` указаны как `optionalDependencies`, поэтому выбор `local-fs` / `smtp` / `webpush` / `pg-trgm` не добавляет никаких затрат на установку.
- **Smoke-тесты** поставляются с каждым адаптером (`backend/scripts/smoke-test-*-providers.ts`) — 27 / 45 / 61 / 55 / 37 утверждений для storage / email / push / search / auth соответственно, все проходят.
- **Полная документация:** см. `backend/docs/providers/` для переменных окружения и настройки каждого провайдера.

## 🎯 Краткое руководство по участию

Начните участвовать **менее чем за 10 минут**:

### Шаг 1: Настройте своё окружение

```bash
# Сделайте форк репозитория на GitHub, затем клонируйте свой форк
git clone https://github.com/YOUR_USERNAME/deskive.git
cd deskive

# Запустите с Docker (самый простой способ)
cp .env.docker .env
docker compose up -d

# Получите доступ к приложению
# Frontend: http://localhost:5175
# Backend API: http://localhost:3000
```

**Вот и всё!** Вы запустили Deskive локально.

### Шаг 2: Найдите то, над чем можно работать

Выберите в зависимости от вашего уровня опыта:

**🟢 Для начинающих**
- 📝 [Исправьте опечатки или улучшите документацию](https://github.com/deskive/deskive/labels/documentation)
- 🌍 [Добавьте переводы](https://github.com/deskive/deskive/labels/i18n) -- Мы поддерживаем английский и японский
- 🐛 [Исправьте простые баги](https://github.com/deskive/deskive/labels/good%20first%20issue)
- ✨ [Улучшите UI/UX](https://github.com/deskive/deskive/labels/ui%2Fux)

**🟡 Средний уровень**
- 🔗 Добавьте новые интеграции -- См. наше [Руководство по интеграции](backend/README.md#integrations)
- 🧪 [Напишите тесты](https://github.com/deskive/deskive/labels/tests)
- 🚀 [Улучшения производительности](https://github.com/deskive/deskive/labels/performance)
- 📱 [Мобильная адаптивность](https://github.com/deskive/deskive/labels/mobile)

**🔴 Продвинутый уровень**
- 🤖 [AI-функции](https://github.com/deskive/deskive/labels/ai) -- Улучшения AutoPilot, новые возможности AI

- 🔐 [Функции безопасности](https://github.com/deskive/deskive/labels/security)

### Шаг 3: Внесите изменения

```bash
# Создайте новую ветку
git checkout -b feature/your-feature-name

# Внесите изменения
# - Код backend: /backend/src/modules
# - Код frontend: /frontend/src
# - Миграции базы данных: /backend/migrations

# Протестируйте свои изменения
npm test

# Сделайте коммит с понятным сообщением
git commit -m "feat: add new integration for XYZ"
```

### Шаг 4: Отправьте свой Pull Request

```bash
# Отправьте в свой форк
git push origin feature/your-feature-name

# Откройте PR на GitHub
# - Опишите, что вы изменили и почему
# - Сошлитесь на связанные проблемы
# - Добавьте скриншоты, если это изменение UI
```

**Что произойдёт дальше?**
- ✅ Автоматические тесты запускаются для вашего PR
- 👀 Мейнтейнер просмотрит ваш код (обычно в течение 48 часов)
- 💬 Мы можем предложить изменения или улучшения
- 🎉 После утверждения ваш код будет слит!
