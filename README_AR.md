## ما هو Deskive؟

Deskive هو **منصة تعاون لمساحات العمل قابلة للاستضافة الذاتية** تجمع بين الاتصال الفوري، وإدارة المشاريع، وأدوات الإنتاجية. مصممة للفرق التي تريد السيطرة الكاملة على بياناتها، يمنحك Deskive وظائف Slack + Notion + Zoom + Asana في تطبيق واحد مفتوح المصدر.

على عكس Slack الذي يتطلب خططًا مدفوعة لمكالمات الفيديو أو Notion الذي يفتقر إلى المحادثة الفورية، يمنحك Deskive كل ما تحتاجه للتعاون بفعالية -- محادثة، مكالمات فيديو، لوحات مشاريع، مشاركة الملفات، مساعدة الذكاء الاصطناعي -- بدون قيود البائع أو الترخيص الاحتكاري.

### كيف يعمل

1. **أنشئ مساحة عملك** -- قم بإعداد مساحات عمل الفريق مع القنوات والمشاريع والأدوار المخصصة
2. **تواصل في الوقت الفعلي** -- دردش مع الردود والتفاعلات والإشارات وصور GIF ومكالمات الفيديو عالية الدقة
3. **إدارة المشاريع** -- نظم العمل باستخدام لوحات Kanban والسباقات وتبعيات المهام وتتبع الوقت
4. **التعاون على المستندات** -- شارك الملاحظات واللوحات البيضاء والملفات مع التحكم في الإصدارات والتوقيعات الرقمية
5. **الأتمتة باستخدام الذكاء الاصطناعي** -- دع AutoPilot يتعامل مع الجدولة وملخصات الاجتماعات والإحاطات اليومية

### القدرات الرئيسية

- **💬 الاتصال الفوري** -- القنوات والرسائل المباشرة والردود والتفاعلات والإشارات ودعم GIF
- **📹 مؤتمرات الفيديو عالية الدقة** -- مكالمات فيديو مدمجة مع مشاركة الشاشة والتسجيل والنسخ عبر LiveKit

- **🤖 مساعد الذكاء الاصطناعي** -- AutoPilot للجدولة وذكاء الاجتماعات وتحليل المستندات

- **🔍 البحث الدلالي** -- بحث مدعوم بالذكاء الاصطناعي عبر جميع أنواع المحتوى
- **🌍 التدويل** -- دعم متعدد اللغات (الإنجليزية واليابانية وقابل للتوسع)

## المشكلة التي نحلها

### معضلة تجزئة أدوات التعاون

تتنقل الفرق الحديثة بين اشتراكات متعددة: Slack للدردشة ($8.75/مستخدم/شهر)، وZoom للفيديو ($15.99/مستخدم/شهر)، وAsana للمشاريع ($10.99/مستخدم/شهر)، وNotion للمستندات ($10/مستخدم/شهر). هذا يخلق سير عمل مجزأ، وصوامع بيانات، ومخاطر أمنية من موردين متعددين، وتكاليف تتوسع خطيًا مع حجم الفريق.

**نقاط الألم الشائعة التي نعالجها:**

- ❌ **تجزئة الأدوات** -- التبديل بين أكثر من 5 أدوات يوميًا يعطل التركيز والإنتاجية
- ❌ **التكاليف المتزايدة** -- اشتراكات SaaS تصل إلى أكثر من $50/مستخدم/شهر للتعاون الأساسي
- ❌ **قفل البيانات** -- بياناتك تعيش على خوادم شخص آخر مع خيارات تصدير محدودة
- ❌ **مخاوف الخصوصية** -- بيانات الأعمال الحساسة تُشارك مع موردين خارجيين متعددين
- ❌ **تعقيد التكامل** -- كل أداة تتطلب تكاملات API منفصلة ومصادقة
- ❌ **فجوات الميزات** -- لا توجد منصة واحدة تقدم ميزات تعاون شاملة

### حل Deskive

✅ **منصة الكل في واحد** -- محادثة، فيديو، مشاريع، ملفات، تقويم، ملاحظات، وذكاء اصطناعي في تطبيق واحد

✅ **استضافة ذاتية ومفتوحة المصدر** -- ملكية كاملة للبيانات مع ترخيص GNU AGPL 3.0

✅ **صفر تكاليف لكل مستخدم** -- تكلفة بنية تحتية واحدة بغض النظر عن حجم الفريق

✅ **تكامل عميق** -- جميع الميزات تتشارك السياق والبيانات بسلاسة

✅ **جاهز للمؤسسات** -- توقيعات رقمية، وسير عمل الموافقات، وسجلات التدقيق، ودعم SSO

### ما الذي يجعل Deskive فريدًا؟

1. **منصة موحدة حقًا** -- جميع الميزات تتشارك نفس نموذج البيانات، مما يتيح تكاملًا عميقًا مستحيلًا مع أدوات منفصلة
2. **استضافة ذاتية بدون تنازلات** -- تكافؤ كامل في الميزات مع بدائل SaaS، بما في ذلك مكالمات الفيديو والذكاء الاصطناعي
3. **مجموعة تقنيات حديثة** -- مبني باستخدام React 19 وNestJS 11 وTypeScript لسهولة الصيانة والأداء
4. **تصميم أصلي للذكاء الاصطناعي** -- بحث متجه، وذاكرة المحادثة، ووكيل AutoPilot مدمج في النظام الأساسي
5. **توسع فعال من حيث التكلفة** -- تكلفة بنية تحتية واحدة تخدم مستخدمين غير محدودين، على عكس تسعير SaaS لكل مقعد

## البدء السريع

### Docker (موصى به)

قم بتشغيل هذه الأوامر من جذر المشروع:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# عدّل .env بتكويناتك (بيانات اعتماد قاعدة البيانات، مفاتيح API، إلخ.)
docker compose up -d
```

هذا كل شيء! يمكنك الوصول إلى التطبيق على `http://localhost:5175` وواجهة برمجة التطبيقات على `http://localhost:3000`.

### الإعداد اليدوي

**المتطلبات المسبقة:** Node.js 20+، PostgreSQL 15+، Redis 7+

```bash
# استنساخ
git clone https://github.com/deskive/deskive.git
cd deskive

# الواجهة الخلفية
cd backend
cp .env.example .env    # عدّل .env بتكويناتك
npm install
npm run migrate         # تشغيل ترحيلات قاعدة البيانات
npm run start:dev

# الواجهة الأمامية (في محطة جديدة)
cd frontend
cp .env.example .env
npm install
npm run dev
```

الواجهة الأمامية: `http://localhost:5175` | الواجهة الخلفية: `http://localhost:3000`

### البدء بأمر واحد

لبيئات التطوير:

```bash
./start.sh
```

**الواجهة الخلفية** (`/backend`) -- NestJS 11 مع TypeScript، PostgreSQL مع استعلامات SQL الخام، Redis للتخزين المؤقت والميزات الفورية، Socket.io لاتصالات WebSocket

**الذكاء الاصطناعي والبحث** -- Qdrant للتضمينات المتجهة، OpenAI لـ GPT-4o-mini ونسخ Whisper

**الفيديو** -- LiveKit لمكالمات الفيديو عالية الدقة، ومشاركة الشاشة، والتسجيل، والنسخ الفوري

## وحدات الميزات

يأتي Deskive مع أكثر من 40 وحدة متكاملة عبر هذه الفئات:

| الفئة | الوحدات |
|----------|---------|
| **الاتصالات** | المحادثة (القنوات، الرسائل المباشرة، الردود)، مكالمات الفيديو (عالية الدقة، التسجيل)، البريد الإلكتروني (Gmail OAuth، SMTP/IMAP)، الإخطارات |
| **إدارة المشاريع** | المهام، المعالم، السباقات، لوحات Kanban، تتبع الوقت، التبعيات، التسميات |
| **المحتوى** | الملاحظات (محرر الكتل)، المستندات (التوقيعات الرقمية)، اللوحات البيضاء، إدارة الملفات (الإصدارات، المشاركة) |
| **الإنتاجية** | التقويم (الأحداث، الغرف)، النماذج (المنشئ، التحليلات)، الموافقات (سير العمل)، الميزانيات (النفقات، الفواتير) |
| **الذكاء الاصطناعي والأتمتة** | AutoPilot (الوكيل)، ذكاء الاجتماعات، تحليل المستندات، البوتات (المشغلات، الإجراءات، الجدولة) |
| **المنصة** | المصادقة (OAuth، SSO)، إدارة مساحة العمل، الأدوار والأذونات، البحث (الدلالي)، التحليلات، التكاملات |

## مزودات قابلة للتبديل

كل خدمة خلفية قابلة للتبديل عبر متغير بيئة واحد. الإعدادات الافتراضية تُبقي Deskive قابلاً للتشغيل بدون أي بيانات اعتماد سحابية؛ انتقل إلى مزود مُدار عندما تكون جاهزاً.

| المجال | متغير البيئة | المزودات المُضمَّنة |
|---|---|---|
| **التخزين** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (افتراضي)، `s3`، `r2`، `minio`، `b2`، `gcs`، `azure`، `none` |
| **البريد الإلكتروني** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`، `resend`، `sendgrid`، `postmark`، `ses`، `mailgun`، `none` |
| **الإشعارات الفورية** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`، `fcm`، `onesignal`، `expo`، `none` |
| **البحث** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (افتراضي، بدون بنية تحتية إضافية)، `meilisearch`، `typesense`، `none` |
| **المصادقة / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`، `google`، `github`، `magic-link` (بدون كلمة مرور، قائم على JWT) |
| **الفيديو** | `VIDEO_PROVIDER` | `livekit`، `jitsi`، `daily`، `agora`، `whereby`، `none` |
| **الذكاء الاصطناعي** | `AI_PROVIDER` | `openai`، `anthropic`، `gemini`، `groq`، `ollama` (محلي) |

- **البحث بالكلمات المفتاحية والدلالي يتعايشان معاً.** يتعامل `SearchProviderService` مع البحث بالكلمات المفتاحية القائم على trigram والمصنَّف؛ بينما يستمر `SearchService` في التعامل مع البحث الدلالي/المتجهي عبر Qdrant.
- **تُحمَّل حزم SDK الاختيارية بشكل كسول** — `@azure/storage-blob` و`@google-cloud/storage` و`firebase-admin` و`livekit-server-sdk` و`agora-token` كلها `optionalDependencies`، لذا اختيار `local-fs` / `smtp` / `webpush` / `pg-trgm` لا يضيف أي تكلفة تثبيت.
- **اختبارات التدخين (Smoke tests)** مرفقة مع كل مُحوِّل (`backend/scripts/smoke-test-*-providers.ts`) — 27 / 45 / 61 / 55 / 37 تأكيداً للتخزين / البريد الإلكتروني / الإشعارات الفورية / البحث / المصادقة على التوالي، وجميعها ناجحة.
- **التوثيق الكامل:** راجع `backend/docs/providers/` لمتغيرات البيئة وإعدادات كل مزود.

## 🚀 لماذا المساهمة في Deskive؟

Deskive هو أكثر من مجرد مشروع مفتوح المصدر آخر -- إنه فرصة لبناء مستقبل التعاون الجماعي أثناء إتقان ممارسات التطوير الحديثة.

### ما ستكسبه

**📚 تعلم مجموعة تقنيات حديثة**
- **React 19** -- أحدث React مع ميزات متزامنة ومكونات الخادم
- **NestJS 11** -- إطار عمل Node.js على مستوى المؤسسات مع حقن التبعية
- **TypeScript في كل مكان** -- كتابة قوية، ودعم IDE أفضل، وأخطاء أقل
- **PostgreSQL + Raw SQL** -- تصميم قاعدة البيانات بدون سحر ORM
- **أنظمة الوقت الفعلي** -- Socket.io لـ WebSockets، Redis لـ pub/sub
- **تكامل الذكاء الاصطناعي** -- تضمينات OpenAI، بحث المتجهات مع Qdrant

**💼 بناء محفظتك**
- ساهم في منصة **جاهزة للإنتاج** تستخدمها الفرق في جميع أنحاء العالم
- اعمل على ميزات تظهر في ملفك الشخصي على GitHub
- احصل على الاعتراف في قاعة مشاهير المساهمين لدينا
- بناء خبرة في **منصات التعاون** و**الأنظمة الفورية** -- مهارات ذات قيمة عالية في 2026

**🤝 انضم إلى مجتمع متنامٍ**
- تواصل مع المطورين من جميع أنحاء العالم
- احصل على مراجعات الكود من المشرفين ذوي الخبرة
- تعلم أفضل الممارسات في معمارية البرمجيات
- شارك في المناقشات التقنية وقرارات التصميم

**🎯 اصنع تأثيرًا حقيقيًا**
- سيساعد كودك الفرق على التحرر من اشتراكات SaaS الباهظة
- شاهد ميزاتك قيد الاستخدام في بيئات الإنتاج
- أثّر على اتجاه أدوات التعاون مفتوحة المصدر

**⚡ إعداد سريع**
- Docker Compose يتيح لك العمل في **أقل من 5 دقائق**
- قاعدة كود موثقة جيدًا مع معمارية واضحة
- مشرفون ودودون يستجيبون لطلبات السحب في غضون 48 ساعة
- تسميات "مشكلة أولى جيدة" للقادمين الجدد

## 🎯 دليل المساهمة السريع

ابدأ المساهمة في **أقل من 10 دقائق**:

### الخطوة 1: إعداد بيئتك

```bash
# انسخ المستودع على GitHub، ثم استنسخ نسختك
git clone https://github.com/YOUR_USERNAME/deskive.git
cd deskive

# ابدأ باستخدام Docker (أسهل طريقة)
cp .env.docker .env
docker compose up -d

# الوصول إلى التطبيق
# الواجهة الأمامية: http://localhost:5175
# واجهة برمجة التطبيقات الخلفية: http://localhost:3000
```

**هذا كل شيء!** أنت الآن تشغل Deskive محليًا.

### الخطوة 2: ابحث عن شيء للعمل عليه

اختر بناءً على مستوى خبرتك:

**🟢 صديق للمبتدئين**
- 📝 [إصلاح الأخطاء المطبعية أو تحسين التوثيق](https://github.com/deskive/deskive/labels/documentation)
- 🌍 [إضافة ترجمات](https://github.com/deskive/deskive/labels/i18n) -- ندعم الإنجليزية واليابانية
- 🐛 [إصلاح الأخطاء البسيطة](https://github.com/deskive/deskive/labels/good%20first%20issue)
- ✨ [تحسين واجهة المستخدم/تجربة المستخدم](https://github.com/deskive/deskive/labels/ui%2Fux)

**🟡 متوسط**
- 🔗 إضافة تكاملات جديدة -- راجع [دليل التكامل](backend/README.md#integrations)
- 🧪 [كتابة الاختبارات](https://github.com/deskive/deskive/labels/tests)
- 🚀 [تحسينات الأداء](https://github.com/deskive/deskive/labels/performance)
- 📱 [استجابة الهاتف المحمول](https://github.com/deskive/deskive/labels/mobile)

**🔴 متقدم**

- 🏗️ [تحسينات المعمارية](https://github.com/deskive/deskive/labels/architecture)
- 🔐 [ميزات الأمان](https://github.com/deskive/deskive/labels/security)

### الخطوة 3: قم بإجراء تغييراتك

```bash
# إنشاء فرع جديد
git checkout -b feature/your-feature-name

# قم بإجراء تغييراتك
# - كود الواجهة الخلفية: /backend/src/modules
# - كود الواجهة الأمامية: /frontend/src
# - ترحيلات قاعدة البيانات: /backend/migrations

# اختبر تغييراتك
npm test

# الالتزام برسالة واضحة
git commit -m "feat: add new integration for XYZ"
```

### الخطوة 4: قدم طلب السحب الخاص بك

```bash
# ادفع إلى نسختك
git push origin feature/your-feature-name

# افتح طلب سحب على GitHub
# - صف ما غيرته ولماذا
# - اربط بأي مشكلات ذات صلة
# - أضف لقطات شاشة إذا كان تغييرًا في واجهة المستخدم
```

**ماذا يحدث بعد ذلك؟**
- ✅ تُشغّل الاختبارات الآلية على طلب السحب الخاص بك
- 👀 يراجع أحد المشرفين كودك (عادةً في غضون 48 ساعة)
- 💬 قد نقترح تغييرات أو تحسينات
- 🎉 بمجرد الموافقة، يتم دمج كودك!

❓ **اطرح الأسئلة** -- افتح [نقاش GitHub](https://github.com/deskive/deskive/discussions) إذا كنت عالقًا
