## 什么是 Deskive？

Deskive 是一个**可自托管的工作空间协作平台**，它将实时通信、项目管理和生产力工具整合在一起。为希望完全掌控数据的团队而打造，Deskive 在一个开源应用程序中为您提供 Slack + Notion + Zoom + Asana 的功能。

与 Slack 需要付费计划才能使用视频通话或 Notion 缺乏实时聊天不同，Deskive 为您提供有效协作所需的一切 -- 聊天、视频通话、项目看板、文件共享、AI 助手 -- 无供应商锁定或专有许可。

### 工作原理

1. **创建您的工作空间** -- 设置团队工作空间，包含频道、项目和自定义角色
2. **实时沟通** -- 支持消息串、表情回应、@提醒、GIF 和高清视频通话
3. **管理项目** -- 使用看板、Sprint、任务依赖和时间跟踪组织工作
4. **协作编辑文档** -- 共享笔记、白板、文件，支持版本控制和数字签名
5. **AI 自动化** -- 让 AutoPilot 处理日程安排、会议摘要和每日简报

### 核心能力

- **💬 实时通信** -- 频道、私信、消息串、表情回应、@提醒和 GIF 支持
- **📹 高清视频会议** -- 内置视频通话，支持屏幕共享、录制和通过 LiveKit 进行转录
- **📋 项目管理** -- 看板、Sprint、里程碑、任务依赖和时间跟踪
- **📁 文件管理** -- 云存储，支持版本控制、共享和 Google Drive 集成
- **📝 协作笔记** -- 基于块的编辑器，支持实时协作和模板
- **📅 日历与日程** -- 事件管理、定期事件、会议室和可用性跟踪
- **🎨 白板** -- 用于头脑风暴和规划的可视化协作工作空间
- **🤖 AI 助手** -- AutoPilot，用于日程安排、会议智能和文档分析
- **📊 表单与分析** -- 自定义表单构建器，支持响应跟踪和工作空间指标
- **✅ 审批流程** -- 文档和流程的内置审批系统
- **💰 预算跟踪** -- 费用管理、计费费率和预算监控
- **🔗 集成** -- 连接 Slack、Google Drive、GitHub、Dropbox 等
- **🔍 语义搜索** -- AI 驱动的跨所有内容类型搜索
- **🌍 国际化** -- 多语言支持（英语、日语，可扩展）

## 我们解决的问题

### 协作工具分散化困境

现代团队需要同时订阅多个服务：Slack 用于聊天（$8.75/用户/月）、Zoom 用于视频（$15.99/用户/月）、Asana 用于项目（$10.99/用户/月）、Notion 用于文档（$10/用户/月）。这导致工作流分散、数据孤岛、多供应商带来的安全风险，以及随团队规模线性增长的成本。

**我们解决的常见痛点：**

- ❌ **工具分散** -- 每天在 5 个以上工具之间切换会破坏专注力和生产力
- ❌ **成本上涨** -- SaaS 订阅费用累计达到 $50+/用户/月仅用于基本协作
- ❌ **数据锁定** -- 您的数据存储在他人的服务器上，导出选项有限
- ❌ **隐私顾虑** -- 敏感业务数据与多个第三方供应商共享
- ❌ **集成复杂** -- 每个工具需要单独的 API 集成和身份验证
- ❌ **功能缺口** -- 没有单一平台提供全面的协作功能

### Deskive 的解决方案

✅ **一体化平台** -- 聊天、视频、项目、文件、日历、笔记和 AI 集成在一个应用程序中

✅ **自托管与开源** -- 完全的数据所有权，采用 GNU AGPL 3.0 许可证

✅ **零按用户收费** -- 无论团队规模如何，只有一项基础设施成本

✅ **深度集成** -- 所有功能无缝共享上下文和数据

✅ **企业级** -- 数字签名、审批工作流、审计日志和 SSO 支持

### 是什么让 Deskive 与众不同？

1. **真正的统一平台** -- 所有功能共享相同的数据模型，实现单独工具无法实现的深度集成
2. **自托管无妥协** -- 与 SaaS 替代方案功能完全对等，包括视频通话和 AI
3. **现代技术栈** -- 使用 React 19、NestJS 11 和 TypeScript 构建，保证可维护性和性能
4. **AI 原生设计** -- 向量搜索、对话记忆和 AutoPilot 代理内置于核心平台
5. **成本效益扩展** -- 一项基础设施成本可服务无限用户，与按席位计费的 SaaS 定价不同

## 快速开始

### Docker（推荐）

从项目根目录运行以下命令：

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# 使用您的配置编辑 .env（数据库凭据、API 密钥等）
docker compose up -d
```

就这样！在 `http://localhost:5175` 访问应用，在 `http://localhost:3000` 访问 API。

### 手动设置

**前置要求：** Node.js 20+、PostgreSQL 15+、Redis 7+

```bash
# 克隆
git clone https://github.com/deskive/deskive.git
cd deskive

# 后端
cd backend
cp .env.example .env    # 使用您的配置编辑 .env
npm install
npm run migrate         # 运行数据库迁移
npm run start:dev

# 前端（在新终端中）
cd frontend
cp .env.example .env
npm install
npm run dev
```

前端：`http://localhost:5175` | 后端：`http://localhost:3000`

### 一键启动

用于开发环境：

```bash
./start.sh
```

## 架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Frontend (React 19)                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │   Chat   │  │ Projects │  │  Files   │  │ Calendar │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│         Vite + TypeScript + Tailwind CSS + Radix UI         │
└────────────────────────┬────────────────────────────────────┘
                         │ REST API + Socket.io
┌────────────────────────┴────────────────────────────────────┐
│                    Backend (NestJS 11)                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │   Auth   │  │   Chat   │  │  Tasks   │  │    AI    │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│         40+ Modules with TypeScript + Raw SQL               │
└────────┬─────────────┬─────────────┬─────────────┬──────────┘
         │             │             │             │
    ┌────┴────┐   ┌────┴────┐  ┌────┴────┐   ┌────┴────┐
    │Postgres │   │  Redis  │  │ Qdrant  │   │LiveKit  │
    │(Storage)│   │(Cache)  │  │(Vector) │   │(Video)  │
    └─────────┘   └─────────┘  └─────────┘   └─────────┘
```

**前端** (`/frontend`) -- React 19 with Vite、TypeScript、Tailwind CSS、Radix UI 组件、Zustand 用于状态管理、React Query 用于数据获取

**后端** (`/backend`) -- NestJS 11 with TypeScript、PostgreSQL 使用原始 SQL 查询、Redis 用于缓存和实时功能、Socket.io 用于 WebSocket 连接

**AI 与搜索** -- Qdrant 用于向量嵌入、OpenAI 用于 GPT-4o-mini 和 Whisper 转录

**视频** -- LiveKit 用于高清视频通话、屏幕共享、录制和实时转录

### 步骤 2：找到要做的事情

根据您的经验水平选择：

**🟢 适合初学者**
- 📝 [修复拼写错误或改进文档](https://github.com/deskive/deskive/labels/documentation)
- 🌍 [添加翻译](https://github.com/deskive/deskive/labels/i18n) -- 我们支持英语和日语
- 🐛 [修复简单的错误](https://github.com/deskive/deskive/labels/good%20first%20issue)
- ✨ [改进 UI/UX](https://github.com/deskive/deskive/labels/ui%2Fux)

**🟡 中级**
- 🔗 添加新集成 -- 查看我们的[集成指南](backend/README.md#integrations)
- 🧪 [编写测试](https://github.com/deskive/deskive/labels/tests)
- 🚀 [性能改进](https://github.com/deskive/deskive/labels/performance)
- 📱 [移动端响应式](https://github.com/deskive/deskive/labels/mobile)

**🔴 高级**
- 🤖 [AI 功能](https://github.com/deskive/deskive/labels/ai) -- AutoPilot 增强、新 AI 功能
- ⚙️ [核心引擎增强](https://github.com/deskive/deskive/labels/core)
- 🏗️ [架构改进](https://github.com/deskive/deskive/labels/architecture)
- 🔐 [安全功能](https://github.com/deskive/deskive/labels/security)

### 步骤 3：进行更改

```bash
# 创建新分支
git checkout -b feature/your-feature-name

# 进行更改
# - 后端代码：/backend/src/modules
# - 前端代码：/frontend/src
# - 数据库迁移：/backend/migrations

# 测试您的更改
npm test

# 使用清晰的消息提交
git commit -m "feat: add new integration for XYZ"
```

### 步骤 4：提交您的 Pull Request

```bash
# 推送到您的 fork
git push origin feature/your-feature-name

# 在 GitHub 上打开 PR
# - 描述您更改了什么以及为什么
# - 链接到任何相关问题
# - 如果是 UI 更改，请添加截图
```

**接下来会发生什么？**
- ✅ 自动化测试在您的 PR 上运行
- 👀 维护者审查您的代码（通常在 48 小时内）
- 💬 我们可能会建议更改或改进
- 🎉 一旦获得批准，您的代码就会被合并！
