## ВҝQuГ© es Deskive?

Deskive es una **plataforma de colaboraciГіn en espacios de trabajo auto-alojable** que reГәne comunicaciГіn en tiempo real, gestiГіn de proyectos y herramientas de productividad. DiseГұada para equipos que desean tener control completo sobre sus datos, Deskive te brinda la funcionalidad de Slack + Notion + Zoom + Asana en una Гәnica aplicaciГіn de cГіdigo abierto.

A diferencia de Slack que requiere planes de pago para videollamadas o Notion que carece de chat en tiempo real, Deskive te proporciona todo lo que necesitas para colaborar efectivamente -- chat, videollamadas, tableros de proyectos, compartir archivos, asistencia de IA -- sin dependencia de proveedores ni licencias propietarias.

### CГіmo Funciona

1. **Crea Tu Espacio de Trabajo** -- Configura espacios de trabajo del equipo con canales, proyectos y roles personalizados
2. **ComunГӯcate en Tiempo Real** -- Chatea con hilos, reacciones, menciones, GIFs y videollamadas en HD
3. **Gestiona Proyectos** -- Organiza el trabajo con tableros Kanban, sprints, dependencias de tareas y seguimiento de tiempo
4. **Colabora en Documentos** -- Comparte notas, pizarras, archivos con control de versiones y firmas digitales
5. **Automatiza con IA** -- AutoPilot tiene acceso completo a toda la aplicaciГіn y puede automatizar cualquier cosa -- programaciГіn, mensajes, actualizaciones de proyectos y mГЎs
6. **ObtГ©n Sugerencias Inteligentes** -- La IA analiza tus datos y sugiere tareas, acciones y prioridades directamente desde el panel

## QuГ© Problema Resolvemos

### El Dilema de la FragmentaciГіn de Herramientas de ColaboraciГіn

Los equipos modernos manejan mГәltiples suscripciones: Slack para chat ($8.75/usuario/mes), Zoom para video ($15.99/usuario/mes), Asana para proyectos ($10.99/usuario/mes), Notion para documentos ($10/usuario/mes). Esto crea flujos de trabajo fragmentados, silos de datos, riesgos de seguridad de mГәltiples proveedores y costos que escalan linealmente con el tamaГұo del equipo.

**Puntos de dolor comunes que abordamos:**

- вқҢ **FragmentaciГіn de Herramientas** -- Cambiar entre mГЎs de 5 herramientas diariamente interrumpe el enfoque y la productividad
- вқҢ **Costos Crecientes** -- Las suscripciones SaaS suman mГЎs de $50/usuario/mes para colaboraciГіn bГЎsica
- вқҢ **Dependencia de Datos** -- Tus datos viven en los servidores de otra persona con opciones limitadas de exportaciГіn
- вқҢ **Preocupaciones de Privacidad** -- Datos comerciales sensibles compartidos con mГәltiples proveedores externos
- вқҢ **Complejidad de IntegraciГіn** -- Cada herramienta requiere integraciones API separadas y autenticaciГіn
- вқҢ **Brechas de CaracterГӯsticas** -- Ninguna plataforma Гәnica ofrece caracterГӯsticas de colaboraciГіn completas

### SoluciГіn de Deskive

вң… **Plataforma Todo en Uno** -- Chat, video, proyectos, archivos, calendario, notas e IA en una aplicaciГіn

вң… **Auto-Alojado y de CГіdigo Abierto** -- Propiedad completa de datos con licencia GNU AGPL 3.0

вң… **Cero Costos por Usuario** -- Un costo de infraestructura independientemente del tamaГұo del equipo

вң… **IntegraciГіn Profunda** -- Todas las caracterГӯsticas comparten contexto y datos sin problemas

вң… **Listo para Empresas** -- Firmas digitales, flujos de aprobaciГіn, registros de auditorГӯa y soporte SSO

## Inicio RГЎpido

### Docker (Recomendado)

Ejecuta estos comandos desde la raГӯz del proyecto:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# Edita .env con tu configuraciГіn (credenciales de base de datos, claves API, etc.)
docker compose up -d
```

ВЎEso es todo! Accede a la aplicaciГіn en `http://localhost:5175` y a la API en `http://localhost:3000`.

### ConfiguraciГіn Manual

**Prerrequisitos:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# Clonar
git clone https://github.com/deskive/deskive.git
cd deskive

# Backend
cd backend
cp .env.example .env    # Edita .env con tu configuraciГіn
npm install
npm run migrate         # Ejecuta migraciones de base de datos
npm run start:dev

# Frontend (en una nueva terminal)
cd frontend
cp .env.example .env
npm install
npm run dev
```

Frontend: `http://localhost:5175` | Backend: `http://localhost:3000`

### Inicio con Un Comando

Para entornos de desarrollo:

```bash
./start.sh
```

## Proveedores Intercambiables

Cada servicio subyacente puede intercambiarse mediante una Гәnica variable de entorno. Los valores por defecto mantienen Deskive ejecutable sin credenciales de nube; cambia a un proveedor gestionado cuando estГ©s listo.

| Dominio | Variable de entorno | Proveedores incluidos |
|---|---|---|
| **Almacenamiento** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (por defecto), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **Email** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **Push** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **BГәsqueda** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (por defecto, sin infraestructura adicional), `meilisearch`, `typesense`, `none` |
| **Auth / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (sin contraseГұa, basado en JWT) |
| **VГӯdeo** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **IA** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (local) |

- **La bГәsqueda por palabras clave y la semГЎntica coexisten.** `SearchProviderService` gestiona la bГәsqueda por trigramas y facetas; `SearchService` sigue encargГЎndose de la bГәsqueda vectorial/semГЎntica con Qdrant.
- **Los SDK opcionales se cargan de forma diferida** вҖ” `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk` y `agora-token` son `optionalDependencies`, por lo que elegir `local-fs` / `smtp` / `webpush` / `pg-trgm` no aГұade ningГәn coste de instalaciГіn.
- **Los smoke tests** se incluyen con cada adaptador (`backend/scripts/smoke-test-*-providers.ts`) вҖ” 27 / 45 / 61 / 55 / 37 aserciones para almacenamiento / email / push / bГәsqueda / auth respectivamente, todas pasan.
- **DocumentaciГіn completa:** consulta `backend/docs/providers/` para las variables de entorno y configuraciГіn de cada proveedor.
