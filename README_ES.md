## ¿Qué es Deskive?

Deskive es una **plataforma de colaboración en espacios de trabajo auto-alojable** que reúne comunicación en tiempo real, gestión de proyectos y herramientas de productividad. Diseñada para equipos que desean tener control completo sobre sus datos, Deskive te brinda la funcionalidad de Slack + Notion + Zoom + Asana en una única aplicación de código abierto.

A diferencia de Slack que requiere planes de pago para videollamadas o Notion que carece de chat en tiempo real, Deskive te proporciona todo lo que necesitas para colaborar efectivamente -- chat, videollamadas, tableros de proyectos, compartir archivos, asistencia de IA -- sin dependencia de proveedores ni licencias propietarias.

### Cómo Funciona

1. **Crea Tu Espacio de Trabajo** -- Configura espacios de trabajo del equipo con canales, proyectos y roles personalizados
2. **Comunícate en Tiempo Real** -- Chatea con hilos, reacciones, menciones, GIFs y videollamadas en HD
3. **Gestiona Proyectos** -- Organiza el trabajo con tableros Kanban, sprints, dependencias de tareas y seguimiento de tiempo
4. **Colabora en Documentos** -- Comparte notas, pizarras, archivos con control de versiones y firmas digitales
5. **Automatiza con IA** -- Deja que AutoPilot maneje la programación, resúmenes de reuniones y actualizaciones diarias

### Capacidades Clave

- **💬 Comunicación en Tiempo Real** -- Canales, mensajes directos, hilos, reacciones, menciones y soporte para GIF
- **📹 Videoconferencia HD** -- Videollamadas integradas con compartir pantalla, grabación y transcripción vía LiveKit
- **📋 Gestión de Proyectos** -- Tableros Kanban, sprints, hitos, dependencias de tareas y seguimiento de tiempo
- **📁 Gestión de Archivos** -- Almacenamiento en la nube con versionado, compartir e integración con Google Drive
- **📝 Notas Colaborativas** -- Editor basado en bloques con colaboración en tiempo real y plantillas
- **📅 Calendario y Programación** -- Gestión de eventos, eventos recurrentes, salas de reuniones y seguimiento de disponibilidad
- **🎨 Pizarra** -- Espacio de colaboración visual para lluvia de ideas y planificación
- **🤖 Asistente de IA** -- AutoPilot para programación, inteligencia de reuniones y análisis de documentos
- **📊 Formularios y Análisis** -- Constructor de formularios personalizado con seguimiento de respuestas y métricas del espacio de trabajo
- **✅ Flujos de Aprobación** -- Sistema de aprobación integrado para documentos y procesos
- **💰 Seguimiento de Presupuesto** -- Gestión de gastos, tarifas de facturación y monitoreo de presupuesto
- **🔗 Integraciones** -- Conéctate con Slack, Google Drive, GitHub, Dropbox y más
- **🔍 Búsqueda Semántica** -- Búsqueda impulsada por IA en todos los tipos de contenido
- **🌍 Internacionalización** -- Soporte multiidioma (inglés, japonés, expandible)

## Qué Problema Resolvemos

### El Dilema de la Fragmentación de Herramientas de Colaboración

Los equipos modernos manejan múltiples suscripciones: Slack para chat ($8.75/usuario/mes), Zoom para video ($15.99/usuario/mes), Asana para proyectos ($10.99/usuario/mes), Notion para documentos ($10/usuario/mes). Esto crea flujos de trabajo fragmentados, silos de datos, riesgos de seguridad de múltiples proveedores y costos que escalan linealmente con el tamaño del equipo.

**Puntos de dolor comunes que abordamos:**

- ❌ **Fragmentación de Herramientas** -- Cambiar entre más de 5 herramientas diariamente interrumpe el enfoque y la productividad
- ❌ **Costos Crecientes** -- Las suscripciones SaaS suman más de $50/usuario/mes para colaboración básica
- ❌ **Dependencia de Datos** -- Tus datos viven en los servidores de otra persona con opciones limitadas de exportación
- ❌ **Preocupaciones de Privacidad** -- Datos comerciales sensibles compartidos con múltiples proveedores externos
- ❌ **Complejidad de Integración** -- Cada herramienta requiere integraciones API separadas y autenticación
- ❌ **Brechas de Características** -- Ninguna plataforma única ofrece características de colaboración completas

### Solución de Deskive

✅ **Plataforma Todo en Uno** -- Chat, video, proyectos, archivos, calendario, notas e IA en una aplicación

✅ **Auto-Alojado y de Código Abierto** -- Propiedad completa de datos con licencia GNU AGPL 3.0

✅ **Cero Costos por Usuario** -- Un costo de infraestructura independientemente del tamaño del equipo

✅ **Integración Profunda** -- Todas las características comparten contexto y datos sin problemas

✅ **Listo para Empresas** -- Firmas digitales, flujos de aprobación, registros de auditoría y soporte SSO

### ¿Qué Hace Único a Deskive?

1. **Plataforma Verdaderamente Unificada** -- Todas las características comparten el mismo modelo de datos, permitiendo una integración profunda imposible con herramientas separadas
2. **Auto-Alojamiento sin Compromisos** -- Paridad completa de características con alternativas SaaS, incluyendo videollamadas e IA

## Inicio Rápido

### Docker (Recomendado)

Ejecuta estos comandos desde la raíz del proyecto:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# Edita .env con tu configuración (credenciales de base de datos, claves API, etc.)
docker compose up -d
```

¡Eso es todo! Accede a la aplicación en `http://localhost:5175` y a la API en `http://localhost:3000`.

### Configuración Manual

**Prerrequisitos:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# Clonar
git clone https://github.com/deskive/deskive.git
cd deskive

# Backend
cd backend
cp .env.example .env    # Edita .env con tu configuración
npm install
npm run migrate         # Ejecuta migraciones de base de datos
npm run start:dev

# Frontend (en una nueva terminal)
cd frontend
cp .env.example .env
npm install
npm run dev
```

Frontend: `http://localhost:5175` | Backend: `http://localhost:3000`

### Inicio con Un Comando

Para entornos de desarrollo:

```bash
./start.sh
```

## Arquitectura

```
┌─────────────────────────────────────────────────────────────┐
│                     Frontend (React 19)                      │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │   Chat   │  │ Proyectos│  │ Archivos │  │Calendario│   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│         Vite + TypeScript + Tailwind CSS + Radix UI         │
└────────────────────────┬────────────────────────────────────┘
                         │ REST API + Socket.io
┌────────────────────────┴────────────────────────────────────┐
│                    Backend (NestJS 11)                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │   Auth   │  │   Chat   │  │  Tareas  │  │    IA    │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│      Más de 40 Módulos con TypeScript + SQL Crudo          │
└────────┬─────────────┬─────────────┬─────────────┬──────────┘
         │             │             │             │
    ┌────┴────┐   ┌────┴────┐  ┌────┴────┐   ┌────┴────┐
    │Postgres │   │  Redis  │  │ Qdrant  │   │LiveKit  │
    │(Almac.) │   │ (Caché) │  │(Vector) │   │ (Video) │
    └─────────┘   └─────────┘  └─────────┘   └─────────┘
```

**Frontend** (`/frontend`) -- React 19 con Vite, TypeScript, Tailwind CSS, componentes Radix UI, Zustand para gestión de estado, React Query para obtención de datos

**Backend** (`/backend`) -- NestJS 11 con TypeScript, PostgreSQL con consultas SQL crudas, Redis para caché y características en tiempo real, Socket.io para conexiones WebSocket

**IA y Búsqueda** -- Qdrant para embeddings vectoriales, OpenAI para GPT-4o-mini y transcripción Whisper

**Video** -- LiveKit para videollamadas HD, compartir pantalla, grabación y transcripción en tiempo real

## Proveedores Intercambiables

Cada servicio subyacente puede intercambiarse mediante una única variable de entorno. Los valores por defecto mantienen Deskive ejecutable sin credenciales de nube; cambia a un proveedor gestionado cuando estés listo.

| Dominio | Variable de entorno | Proveedores incluidos |
|---|---|---|
| **Almacenamiento** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (por defecto), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **Email** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **Push** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **Búsqueda** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (por defecto, sin infraestructura adicional), `meilisearch`, `typesense`, `none` |
| **Auth / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (sin contraseña, basado en JWT) |
| **Vídeo** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **IA** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (local) |

- **La búsqueda por palabras clave y la semántica coexisten.** `SearchProviderService` gestiona la búsqueda por trigramas y facetas; `SearchService` sigue encargándose de la búsqueda vectorial/semántica con Qdrant.
- **Los SDK opcionales se cargan de forma diferida** — `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk` y `agora-token` son `optionalDependencies`, por lo que elegir `local-fs` / `smtp` / `webpush` / `pg-trgm` no añade ningún coste de instalación.
- **Los smoke tests** se incluyen con cada adaptador (`backend/scripts/smoke-test-*-providers.ts`) — 27 / 45 / 61 / 55 / 37 aserciones para almacenamiento / email / push / búsqueda / auth respectivamente, todas pasan.
- **Documentación completa:** consulta `backend/docs/providers/` para las variables de entorno y configuración de cada proveedor.

- Etiquetas de "good first issue" para principiantes

## 🎯 Guía Rápida de Contribución

Comienza a contribuir en **menos de 10 minutos**:

### Paso 1: Configura Tu Entorno

```bash
# Haz fork del repositorio en GitHub, luego clona tu fork
git clone https://github.com/TU_USUARIO/deskive.git
cd deskive

# Inicia con Docker (forma más fácil)
cp .env.docker .env
docker compose up -d

# Accede a la aplicación
# Frontend: http://localhost:5175
# API Backend: http://localhost:3000
```

**¡Eso es todo!** Estás ejecutando Deskive localmente.

### Paso 2: Encuentra Algo en Qué Trabajar

Elige según tu nivel de experiencia:

**🟢 Apto para Principiantes**
- 📝 [Corrige errores tipográficos o mejora la documentación](https://github.com/deskive/deskive/labels/documentation)
- 🌍 [Agrega traducciones](https://github.com/deskive/deskive/labels/i18n) -- Soportamos inglés y japonés
- 🐛 [Corrige errores simples](https://github.com/deskive/deskive/labels/good%20first%20issue)
- ✨ [Mejora UI/UX](https://github.com/deskive/deskive/labels/ui%2Fux)

**🟡 Intermedio**
- 🔗 Agrega nuevas integraciones -- Consulta nuestra [Guía de Integración](backend/README.md#integrations)
- 🧪 [Escribe pruebas](https://github.com/deskive/deskive/labels/tests)
- 🚀 [Mejoras de rendimiento](https://github.com/deskive/deskive/labels/performance)
- 📱 [Capacidad de respuesta móvil](https://github.com/deskive/deskive/labels/mobile)

**🔴 Avanzado**
- 🤖 [Características de IA](https://github.com/deskive/deskive/labels/ai) -- Mejoras de AutoPilot, nuevas capacidades de IA
- ⚙️ [Mejoras del motor central](https://github.com/deskive/deskive/labels/core)
- 🏗️ [Mejoras de arquitectura](https://github.com/deskive/deskive/labels/architecture)
- 🔐 [Características de seguridad](https://github.com/deskive/deskive/labels/security)

### Paso 3: Realiza Tus Cambios

```bash
# Crea una nueva rama
git checkout -b feature/nombre-de-tu-caracteristica

# Realiza tus cambios
# - Código backend: /backend/src/modules
# - Código frontend: /frontend/src
# - Migraciones de base de datos: /backend/migrations

# Prueba tus cambios
npm test

# Haz commit con un mensaje claro
git commit -m "feat: agregar nueva integración para XYZ"
```

### Paso 4: Envía Tu Pull Request

```bash
# Sube a tu fork
git push origin feature/nombre-de-tu-caracteristica

# Abre un PR en GitHub
# - Describe qué cambiaste y por qué
# - Enlaza a cualquier problema relacionado
# - Agrega capturas de pantalla si es un cambio de UI
```

**¿Qué sucede después?**
- ✅ Las pruebas automatizadas se ejecutan en tu PR
- 👀 Un mantenedor revisa tu código (generalmente dentro de 48 horas)
- 💬 Podemos sugerir cambios o mejoras
- 🎉 ¡Una vez aprobado, tu código se fusiona!

### Consejos de Contribución

✨ **Comienza pequeño** -- Tu primer PR no necesita ser una característica enorme

📖 **Lee el código** -- Navega por los módulos existentes en `backend/src/modules` como referencia

❓ **Haz preguntas** -- Abre una [Discusión en GitHub](https://github.com/deskive/deskive/discussions) si estás atascado

🧪 **Escribe pruebas** -- Los PRs con pruebas se fusionan más rápido

📝 **Documenta tu código** -- Agrega comentarios para lógica compleja
