## Deskive क्या है?

Deskive एक **स्व-होस्ट करने योग्य कार्यस्थल सहयोग प्लेटफॉर्म** है जो रियल-टाइम संचार, परियोजना प्रबंधन और उत्पादकता उपकरण को एक साथ लाता है। उन टीमों के लिए बनाया गया जो अपने डेटा पर पूर्ण नियंत्रण चाहते हैं, Deskive आपको एक ही ओपन-सोर्स एप्लिकेशन में Slack + Notion + Zoom + Asana की कार्यक्षमता देता है।

Slack के विपरीत जो वीडियो कॉल के लिए भुगतान योजनाओं की आवश्यकता होती है या Notion में रियल-टाइम चैट की कमी होती है, Deskive आपको प्रभावी ढंग से सहयोग करने के लिए आवश्यक सब कुछ देता है -- चैट, वीडियो कॉल, प्रोजेक्ट बोर्ड, फाइल शेयरिंग, AI सहायता -- बिना किसी vendor lock-in या मालिकाना लाइसेंसिंग के।

### यह कैसे काम करता है

1. **अपना कार्यस्थल बनाएं** -- चैनल, परियोजनाओं और कस्टम भूमिकाओं के साथ टीम कार्यस्थल सेटअप करें
2. **रियल-टाइम में संवाद करें** -- थ्रेड्स, रिएक्शन, मेंशन, GIF और HD वीडियो कॉल के साथ चैट करें
3. **परियोजनाओं का प्रबंधन करें** -- Kanban बोर्ड, स्प्रिंट, कार्य निर्भरता और समय ट्रैकिंग के साथ काम को व्यवस्थित करें
4. **दस्तावेजों पर सहयोग करें** -- संस्करण नियंत्रण और डिजिटल हस्ताक्षर के साथ नोट्स, व्हाइटबोर्ड, फाइलें साझा करें
5. **AI के साथ स्वचालित करें** -- AutoPilot के पास पूरे ऐप पर पूर्ण पहुंच है और यह सब कुछ स्वचालित कर सकता है -- शेड्यूलिंग, संदेश, प्रोजेक्ट अपडेट और बहुत कुछ
6. **स्मार्ट सुझाव प्राप्त करें** -- AI आपके डेटा का विश्लेषण करता है और डैशबोर्ड से सीधे कार्य, कार्रवाई और प्राथमिकताएं सुझाता है

### प्रमुख क्षमताएं

- **💬 रियल-टाइम संचार** -- चैनल, डायरेक्ट मैसेज, थ्रेड्स, रिएक्शन, मेंशन और GIF समर्थन
- **📹 HD वीडियो कॉन्फ्रेंसिंग** -- LiveKit के माध्यम से स्क्रीन शेयरिंग, रिकॉर्डिंग और ट्रांसक्रिप्शन के साथ अंतर्निहित वीडियो कॉल
- **📋 परियोजना प्रबंधन** -- Kanban बोर्ड, स्प्रिंट, मील के पत्थर, कार्य निर्भरता और समय ट्रैकिंग
- **📁 फाइल प्रबंधन** -- वर्जनिंग, शेयरिंग और Google Drive एकीकरण के साथ क्लाउड स्टोरेज
- **📝 सहयोगी नोट्स** -- रियल-टाइम सहयोग और टेम्पलेट्स के साथ ब्लॉक-आधारित एडिटर
- **📅 कैलेंडर और शेड्यूलिंग** -- इवेंट प्रबंधन, आवर्ती इवेंट्स, मीटिंग रूम और उपलब्धता ट्रैकिंग
- **🎨 व्हाइटबोर्ड** -- विचार-मंथन और योजना बनाने के लिए दृश्य सहयोग कार्यस्थल
- **🤖 AutoPilot AI एजेंट** -- पूरे ऐप तक पहुंच वाला पूर्णतः स्वायत्त AI सहायक -- कार्यों को स्वचालित करें, मीटिंग शेड्यूल करें, संदेश भेजें, प्रोजेक्ट प्रबंधित करें और सभी मॉड्यूल में वर्कफ़्लो संभालें
- **🧠 स्मार्ट सुझाव** -- AI-संचालित डैशबोर्ड सुझाव जो आपकी गतिविधि, प्रोजेक्ट और समय सीमा का विश्लेषण करके कार्य और प्राथमिकताएं सुझाते हैं
- **🧰 अंतर्निहित उपकरण** -- दैनिक कार्यों के लिए तैयार उत्पादकता उपकरण -- पोल, रिमाइंडर, समय ट्रैकिंग, टेम्पलेट और अधिक -- कोई अतिरिक्त सेटअप आवश्यक नहीं
- **🔌 कनेक्टर** -- Slack, Google Drive, GitHub, Dropbox, Gmail और अधिक सहित 6+ पूर्व-कॉन्फ़िगर OAuth एकीकरण के साथ 180+ तृतीय-पक्ष ऐप कनेक्टर
- **📊 फॉर्म और विश्लेषण** -- प्रतिक्रिया ट्रैकिंग और कार्यस्थल मेट्रिक्स के साथ कस्टम फॉर्म बिल्डर
- **✅ अनुमोदन वर्कफ़्लो** -- दस्तावेजों और प्रक्रियाओं के लिए अंतर्निहित अनुमोदन प्रणाली
- **💰 बजट ट्रैकिंग** -- व्यय प्रबंधन, बिलिंग दरें और बजट निगरानी
- **🔍 सिमेंटिक खोज** -- सभी सामग्री प्रकारों में AI-संचालित खोज
- **🌍 अंतर्राष्ट्रीयकरण** -- बहु-भाषा समर्थन (अंग्रेजी, जापानी, विस्तार योग्य)

## हम किस समस्या का समाधान करते हैं

### सहयोग उपकरण विखंडन की दुविधा

आधुनिक टीमें कई सब्सक्रिप्शन जुगल करती हैं: चैट के लिए Slack ($8.75/उपयोगकर्ता/माह), वीडियो के लिए Zoom ($15.99/उपयोगकर्ता/माह), परियोजनाओं के लिए Asana ($10.99/उपयोगकर्ता/माह), डॉक्स के लिए Notion ($10/उपयोगकर्ता/माह)। यह खंडित वर्कफ़्लो, डेटा साइलो, कई विक्रेताओं से सुरक्षा जोखिम और टीम के आकार के साथ रैखिक रूप से बढ़ने वाली लागत बनाता है।

**सामान्य समस्याएं जिन्हें हम संबोधित करते हैं:**

- ❌ **उपकरण विखंडन** -- दैनिक रूप से 5+ उपकरणों के बीच स्विच करना फोकस और उत्पादकता को बाधित करता है
- ❌ **बढ़ती लागत** -- बुनियादी सहयोग के लिए SaaS सब्सक्रिप्शन $50+/उपयोगकर्ता/माह तक जोड़ती हैं
- ❌ **डेटा लॉक-इन** -- आपका डेटा सीमित निर्यात विकल्पों के साथ किसी और के सर्वर पर रहता है
- ❌ **गोपनीयता चिंताएं** -- कई तृतीय-पक्ष विक्रेताओं के साथ साझा किया गया संवेदनशील व्यावसायिक डेटा
- ❌ **एकीकरण जटिलता** -- प्रत्येक उपकरण के लिए अलग API एकीकरण और प्रमाणीकरण की आवश्यकता होती है
- ❌ **फीचर अंतराल** -- कोई भी एकल प्लेटफॉर्म व्यापक सहयोग सुविधाएं प्रदान नहीं करता है

### Deskive का समाधान

✅ **ऑल-इन-वन प्लेटफॉर्म** -- एक एप्लिकेशन में चैट, वीडियो, परियोजनाएं, फाइलें, कैलेंडर, नोट्स और AI

✅ **स्व-होस्टेड और ओपन सोर्स** -- GNU AGPL 3.0 लाइसेंस के साथ पूर्ण डेटा स्वामित्व

✅ **शून्य प्रति-उपयोगकर्ता लागत** -- टीम के आकार की परवाह किए बिना एक बुनियादी ढांचा लागत

✅ **गहरा एकीकरण** -- सभी सुविधाएं संदर्भ और डेटा को सहजता से साझा करती हैं

✅ **एंटरप्राइज-रेडी** -- डिजिटल हस्ताक्षर, अनुमोदन वर्कफ़्लो, ऑडिट लॉग और SSO समर्थन

### Deskive को अद्वितीय क्या बनाता है?

1. **सही मायने में एकीकृत प्लेटफॉर्म** -- सभी सुविधाएं एक ही डेटा मॉडल साझा करती हैं, जो अलग उपकरणों के साथ असंभव गहरे एकीकरण को सक्षम करती हैं
2. **समझौता किए बिना स्व-होस्टिंग** -- वीडियो कॉल और AI सहित SaaS विकल्पों के साथ पूर्ण फीचर समानता
3. **आधुनिक टेक स्टैक** -- रखरखाव और प्रदर्शन के लिए React 19, NestJS 11 और TypeScript के साथ निर्मित
4. **AI-नेटिव डिज़ाइन** -- कोर प्लेटफॉर्म में वेक्टर खोज, वार्तालाप मेमोरी और AutoPilot एजेंट शामिल है
5. **लागत-प्रभावी स्केलिंग** -- एक बुनियादी ढांचा लागत असीमित उपयोगकर्ताओं की सेवा करती है, प्रति-सीट SaaS मूल्य निर्धारण के विपरीत

## त्वरित शुरुआत

### Docker (अनुशंसित)

प्रोजेक्ट रूट से ये कमांड चलाएं:

```bash
git clone https://github.com/deskive/deskive.git
cd deskive
cp .env.docker .env
# अपनी कॉन्फ़िगरेशन के साथ .env संपादित करें (डेटाबेस क्रेडेंशियल्स, API कुंजियां, आदि।)
docker compose up -d
```

बस इतना ही! ऐप को `http://localhost:5175` पर और API को `http://localhost:3000` पर एक्सेस करें।

### मैनुअल सेटअप

**पूर्वापेक्षाएँ:** Node.js 20+, PostgreSQL 15+, Redis 7+

```bash
# क्लोन करें
git clone https://github.com/deskive/deskive.git
cd deskive

# बैकएंड
cd backend
cp .env.example .env    # अपनी कॉन्फ़िगरेशन के साथ .env संपादित करें
npm install
npm run migrate         # डेटाबेस माइग्रेशन चलाएं
npm run start:dev

# फ्रंटएंड (एक नए टर्मिनल में)
cd frontend
cp .env.example .env
npm install
npm run dev
```

फ्रंटएंड: `http://localhost:5175` | बैकएंड: `http://localhost:3000`

### एक-कमांड शुरुआत

विकास वातावरण के लिए:

```bash
./start.sh
```

## प्लगेबल प्रोवाइडर

प्रत्येक बैकिंग सेवा एक ही एनवायरनमेंट वेरिएबल के माध्यम से बदली जा सकती है। डिफ़ॉल्ट सेटिंग्स Deskive को बिना किसी क्लाउड क्रेडेंशियल्स के चलाने योग्य बनाए रखती हैं; जब आप तैयार हों तो मैनेज्ड प्रोवाइडर पर स्विच करें।

| डोमेन | एनवायरनमेंट वेरिएबल | उपलब्ध प्रोवाइडर |
|---|---|---|
| **स्टोरेज** (PR [#28](https://github.com/deskive/deskive/pull/28)) | `STORAGE_PROVIDER` | `local-fs` (डिफ़ॉल्ट), `s3`, `r2`, `minio`, `b2`, `gcs`, `azure`, `none` |
| **ईमेल** (PR [#30](https://github.com/deskive/deskive/pull/30)) | `EMAIL_PROVIDER` | `smtp`, `resend`, `sendgrid`, `postmark`, `ses`, `mailgun`, `none` |
| **पुश** (PR [#31](https://github.com/deskive/deskive/pull/31)) | `PUSH_PROVIDER` | `webpush`, `fcm`, `onesignal`, `expo`, `none` |
| **खोज** (PR [#32](https://github.com/deskive/deskive/pull/32)) | `SEARCH_PROVIDER` | `pg-trgm` (डिफ़ॉल्ट, कोई अतिरिक्त इन्फ्रास्ट्रक्चर नहीं), `meilisearch`, `typesense`, `none` |
| **Auth / SSO** (PR [#33](https://github.com/deskive/deskive/pull/33)) | `AUTH_PROVIDERS` | `local`, `google`, `github`, `magic-link` (पासवर्ड रहित, JWT आधारित) |
| **वीडियो** | `VIDEO_PROVIDER` | `livekit`, `jitsi`, `daily`, `agora`, `whereby`, `none` |
| **AI** | `AI_PROVIDER` | `openai`, `anthropic`, `gemini`, `groq`, `ollama` (लोकल) |

- **कीवर्ड और सिमेंटिक खोज साथ-साथ चलते हैं।** `SearchProviderService` ट्राइग्राम/फ़ेसेटेड कीवर्ड खोज को संभालता है; `SearchService` Qdrant वेक्टर/सिमेंटिक खोज को संभालना जारी रखता है।
- **वैकल्पिक SDK आवश्यकता पड़ने पर लोड होते हैं** — `@azure/storage-blob`, `@google-cloud/storage`, `firebase-admin`, `livekit-server-sdk`, और `agora-token` `optionalDependencies` हैं, इसलिए `local-fs` / `smtp` / `webpush` / `pg-trgm` चुनने पर कोई इंस्टॉल लागत नहीं आती।
- **स्मोक टेस्ट** प्रत्येक एडॉप्टर के साथ उपलब्ध हैं (`backend/scripts/smoke-test-*-providers.ts`) — स्टोरेज / ईमेल / पुश / खोज / ऑथ के लिए क्रमशः 27 / 45 / 61 / 55 / 37 एसर्शन, सभी पास हो रहे हैं।
- **पूर्ण दस्तावेज़ीकरण:** प्रति-प्रोवाइडर एनवायरनमेंट वेरिएबल्स और सेटअप के लिए `backend/docs/providers/` देखें।

## 🚀 Deskive में योगदान क्यों करें?

Deskive सिर्फ एक और ओपन-सोर्स प्रोजेक्ट से अधिक है -- यह आधुनिक विकास प्रथाओं में महारत हासिल करते हुए टीम सहयोग के भविष्य को बनाने का एक अवसर है।

### आपको क्या मिलेगा

**📚 आधुनिक टेक स्टैक सीखें**
- **React 19** -- समवर्ती सुविधाओं और सर्वर घटकों के साथ नवीनतम React
- **NestJS 11** -- निर्भरता इंजेक्शन के साथ एंटरप्राइज-ग्रेड Node.js फ्रेमवर्क
- **TypeScript पूरे में** -- मजबूत टाइपिंग, बेहतर IDE समर्थन, कम बग
- **PostgreSQL + रॉ SQL** -- ORM जादू के बिना डेटाबेस डिज़ाइन
- **रियल-टाइम सिस्टम** -- WebSockets के लिए Socket.io, pub/sub के लिए Redis
- **AI एकीकरण** -- OpenAI एम्बेडिंग, Qdrant के साथ वेक्टर खोज

**💼 अपना पोर्टफोलियो बनाएं**
- दुनिया भर की टीमों द्वारा उपयोग किए जाने वाले **उत्पादन-तैयार** प्लेटफॉर्म में योगदान करें
- ऐसी सुविधाओं पर काम करें जो आपके GitHub प्रोफाइल पर दिखाई देती हैं
- हमारे योगदानकर्ता हॉल ऑफ फेम में मान्यता प्राप्त करें
- **सहयोग प्लेटफॉर्म** और **रियल-टाइम सिस्टम** में विशेषज्ञता बनाएं -- 2026 में अत्यधिक मूल्यवान कौशल

**🤝 एक बढ़ते समुदाय में शामिल हों**
- दुनिया भर के डेवलपर्स के साथ जुड़ें
- अनुभवी रखरखावकर्ताओं से कोड समीक्षा प्राप्त करें
- सॉफ्टवेयर आर्किटेक्चर में सर्वोत्तम प्रथाएं सीखें
- तकनीकी चर्चाओं और डिज़ाइन निर्णयों में भाग लें

**🎯 वास्तविक प्रभाव डालें**
- आपका कोड टीमों को महंगी SaaS सब्सक्रिप्शन से मुक्त होने में मदद करेगा
- उत्पादन वातावरण में उपयोग की जा रही अपनी सुविधाओं को देखें
- ओपन-सोर्स सहयोग उपकरणों की दिशा को प्रभावित करें

**⚡ त्वरित ऑनबोर्डिंग**

### चरण 3: अपने परिवर्तन करें

```bash
# एक नई ब्रांच बनाएं
git checkout -b feature/your-feature-name

# अपने परिवर्तन करें
# - बैकएंड कोड: /backend/src/modules
# - फ्रंटएंड कोड: /frontend/src
# - डेटाबेस माइग्रेशन: /backend/migrations

# अपने परिवर्तनों का परीक्षण करें
npm test

# स्पष्ट संदेश के साथ कमिट करें
git commit -m "feat: add new integration for XYZ"
```

### चरण 4: अपना Pull Request सबमिट करें

```bash
# अपने फोर्क पर पुश करें
git push origin feature/your-feature-name

# GitHub पर PR खोलें
# - वर्णन करें कि आपने क्या बदला और क्यों
# - किसी भी संबंधित मुद्दों से लिंक करें
# - यदि यह UI परिवर्तन है तो स्क्रीनशॉट जोड़ें
```

**आगे क्या होता है?**
- ✅ आपके PR पर स्वचालित टेस्ट चलते हैं
- 👀 एक रखरखावकर्ता आपके कोड की समीक्षा करता है (आमतौर पर 48 घंटों के भीतर)
- 💬 हम परिवर्तन या सुधार सुझा सकते हैं
- 🎉 एक बार स्वीकृत होने के बाद, आपका कोड मर्ज हो जाता है!

### योगदान सुझाव

✨ **छोटे से शुरू करें** -- आपके पहले PR को एक बड़ी सुविधा होने की आवश्यकता नहीं है

📖 **कोड पढ़ें** -- संदर्भ के लिए `backend/src/modules` में मौजूदा मॉड्यूल ब्राउज़ करें

❓ **प्रश्न पूछें** -- यदि आप फंस गए हैं तो [GitHub Discussion](https://github.com/deskive/deskive/discussions) खोलें

🧪 **टेस्ट लिखें** -- टेस्ट वाले PR तेजी से मर्ज होते हैं

📝 **अपने कोड को डॉक्यूमेंट करें** -- जटिल तर्क के लिए टिप्पणियां जोड़ें
